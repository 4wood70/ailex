**AILEX 4.19**

Методическая конституция приложения для изучения иностранных языков

*Редакция 4.17/4.19 от 23.06.26 · на базе AILEX 2.0 · синхронизировано с реализованным билдом (AILEX Hybrid 4.14; раздел 6 «Суперквест» — с билдом v31 основной версии)*

# О документе

Это «конституция» методики AILEX — описание логики обучения, которая закладывается в код приложения жёстко и не зависит от версии используемой модели Gemini. Gemini выступает только генератором контента (слова, фразы, проверка ответов, интерактивные рассказы-квесты), а вся последовательность и правила обучения принадлежат приложению.

**Ключевой принцип разделения ответственности:** логика → приложение, контент → Gemini, хранилище → локальная база на устройстве (облако на Google Drive — будущий этап). Смена версии Gemini не должна менять методику.

## Архитектура (общая картина)

- Платформа: Android-приложение (публикация в Google Play). Устройство поддерживает несколько профилей (раздел 2.8): один профиль = одна языковая пара «родной → целевой». Это позволяет учить несколько языков (по профилю на язык) и использовать приложение нескольким людям на одном устройстве. У каждого профиля свои словарь, свод правил, прогресс и аналитика — полностью раздельно.

- Логика обучения: конечный автомат (state machine) внутри приложения — профиль, placement-тест, уроки, чекпоинт/экзамен, режим работы над ошибками.

- Контент: запросы к Gemini API с жёсткими рамками (уровень CEFR, обязательная лексика, длина, запрет оффтопа).

- Хранилище (реализованный билд): локальная база на устройстве AILEX_StatusDB (структура из пяти частей — «Профиль», «Прогресс», «Словарь», «Свод правил» и «Аналитика»); перенос между устройствами — вручную экспортом/импортом JSON. Без входа в Google и без OAuth. Облачный вариант (Google Sheets на Drive) — цель будущего этапа.

- Ключ Gemini: вводится самим пользователем через мастер получения ключа (раздел 0); каждый пользователь использует и оплачивает свой ключ.

## Технология реализации (как собрана основная версия)

Основная (текущая) версия AILEX реализована веб-средствами: HTML, CSS и JavaScript, собрана одним самостоятельным файлом index.html. Файл оформлен как PWA: добавлены manifest.webmanifest, сервис-воркер sw.js и иконки. Хранение прогресса — локальное на устройстве; запросы к Gemini идут напрямую с устройства по ключу пользователя.

**Где лежат исходники:** репозиторий https://github.com/4wood70/ailex, опубликован на GitHub Pages. APK собирается через PWABuilder (TWA). Обновление кода: заменить файлы в репозитории — пересборка APK не нужна.

# 0. Техническая подготовка и доступ

- При первом запуске вход в Google-аккаунт не требуется. Прогресс хранится локально на устройстве (раздел 2.0).

- Если ключ Gemini ещё не задан, приложение запускает мастер получения ключа.

- **Автосохранение без участия пользователя:** запись прогресса выполняется автоматически по завершении этапа.

## Мастер получения ключа Gemini

Цель: превратить «найди и скопируй API-ключ» в простой пошаговый помощник. Делается один раз; дальше ключ хранится в приложении.

- Экран объясняет: «Чтобы AILEX мог учить вас, нужен бесплатный ключ доступа к Gemini. Это займёт минуту».
- Кнопка «Получить ключ» открывает Google AI Studio.
- Ключ создаётся в один-два тапа. Приложение предлагает вставить его в поле.
- Ключ сохраняется в защищённом хранилище устройства.

# 1. Академический фокус (Topic Lockdown)

- **Роль:** AILEX — учитель иностранных языков по стандарту CEFR. Стиль: строгий, лаконичный, дисциплинированный наставник.

- **Жёсткий фокус:** обсуждаются только темы обучения. На оффтоп — Pivot-метод.

- **Язык общения:** все объяснения, комментарии и служебные тексты — строго на родном языке пользователя.

- **Управление состоянием обучения принадлежит приложению**, а не модели.

## Язык интерфейса и пояснений

Два слоя:

- **Служебный интерфейс** (кнопки, названия экранов) — из заготовленной локализации. Показывается мгновенно и без интернета.

- **Учебные пояснения от Gemini** (разбор ошибок, правила, подсказки) — генерируются на родном языке через параметр native_lang контракта.

### Откуда берётся язык интерфейса (три ступени)

- Первый запуск: язык из системных настроек Android.
- После онбординга: «родной язык» из профиля.
- Нет заготовки: автоперевод через Gemini при первом запуске + кэш; запасной — английский.

## Автозавершение этапа (Auto-Save Control)

Явное согласие на сохранение убрано. При достижении финала этапа приложение немедленно и автоматически сохраняет прогресс. Показывается уведомление «Сохранено ✓», которое само исчезает.

# 2. Архитектура данных и памяти

## 2.0. Хранилище данных

**Статус реализации.** Данные хранятся локально на устройстве — база AILEX_StatusDB, без входа в Google и без OAuth. Перенос — вручную через экспорт/импорт файла AILEX_StatusDB.json. Облачный вариант (Google Sheets на Drive) — будущий этап.

## 2.1. Логика индексации

- Индекс урока: [Уровень].[Подуровень].[Номер], например A1.1.01; B2.2.05.
- Уровень A0 (Pre-A1): коды A0.1.01…A0.1.NN (подуровень всегда 1, экзамена нет — раздел 4.3).

## 2.2. Лист «Профиль» — статика

Состав (6 полей): Имя, Родной язык, Целевой язык, Возраст, Интересы / сфера деятельности, Уровень CEFR (определяется placement-тестом; может быть A0).

## 2.3. Лист «Прогресс» — динамика

Назначение: история обучения — лента событий. Рекомендуемые столбцы: Код этапа, Дата, Слова этапа, Следующий этап, Примечание.

### Сценарии записи

- Стандартный урок пройден: A1.1.05 | 01.03.26 | слово1, слово2 | A1.1.06
- Урок A0 пройден: A0.1.03 | 22.06.26 | (фонема) | A0.1.04
- Завершение A0: A0.1.NN | 22.06.26 | (пусто) | A1.1.01
- Суперквест пройден: SUPER_A1.1.05 | 01.03.26 | предлог1 | A1.1.06
- Экзамен сдан: EXAM_A1.1_PASS | 01.03.26 | (пусто) | A1.2.01
- Экзамен провален (1-я попытка): EXAM_A1.1_FAIL-1 | 01.03.26 | (пусто) | EXAM_A1.1
- REMEDIAL: EXAM_A1.1_REMEDIAL | 01.03.26 | ошибка1 | EXAM_A1.1

## 2.3-С. Лист «Словарь»

Полноценный личный словарь — по одной строке на слово (KNOWN_VOCAB).

Поля: Слово (целевой язык), Перевод (родной), Транскрипция (родными буквами), Грамматические формы («паспорт»), Эмодзи-якорь, Неправильный глагол (да/нет), Урок-источник, Число ошибок, Статус освоения, Мнемоника, Даты.

**Прогресс-индикатор:** Большое кольцо — прогресс по ТЕКУЩЕМУ уровню CEFR (по пройденным урокам: 50 уроков на уровень A1+; для A0 — своя шкала, 100% после последнего урока A0). Мелкая подпись — точность ответов (accuracy_first_try). Кольцо A0 обнуляется при переходе на A1.

**Каталог словаря по буквам:** индекс по первой букве самого слова без ведущего артикля (der Bus → «B»). Фильтр «только неправильные глаголы».

## 2.3-М. Мнемотехники

Генерируются один раз по триггеру для трудных слов/правил/фраз, сохраняются в словарь. Три уровня деградации: (1) текст + эмодзи (реализовано), (2) схемы из данных, (3) Imagen-картинка (будущие этапы).

## 2.3-П. Часть «Свод правил»

Личный справочник грамматики. Каждое правило, показанное как Grammar Card, сохраняется со статусом «изучено». Доступен с главного экрана без запуска урока. Группировка по категориям: Артикли, Существительные, Глаголы, Времена, Падежи, Предлоги, Порядок слов, Местоимения, Прочее.

## 2.4. Разделение ответственности: кто пишет в базу

**Железное правило:** Gemini не знает о существовании базы и никогда не пишет в неё. Всю работу с базой выполняет исключительно приложение.

## 2.5. Контракт вызова Gemini (строгий JSON)

Ответ Gemini — всегда строго структурированный JSON, без свободного текста вне полей.

## 2.6. Профиль успеваемости (лист «Аналитика»)

Метрики: точность урока (accuracy_first_try), ошибки по типам, трудные слова, скорость и подсказки, динамика. Пишутся автоматически по итогу каждого урока. Пример строки: { "lesson": "A1.1.05", "date": "01.03.26", "accuracy_first_try": 0.86, "errors_by_type": { "recognition": 0, "writing": 2, "construction": 1, "quest": 0 }, "hard_words": ["messen"], "duration_sec": 540, "hints_used": 3, "delta_accuracy": "+0.05" }

## 2.7. Движок адаптации: «Gemini советует — приложение решает»

Трёхуровневая схема: (1) сбор метрик, (2) диагностика Gemini (строгий JSON из словаря категорий), (3) действие по жёстким правилам приложения.

Словарь диагнозов: dominant_error_type (recognition / spelling / verb_conjugation / word_order / vocabulary_gap / none), risk_words, confidence_trend (improving / stable / declining), note_native.

Пример правил: accuracy < 0.70 → вперёд + усиленное повторение; dominant_error_type = verb_conjugation → добавить блок на спряжение; слово с 2+ ошибками → Повторение следующего урока.

Защита от сбоя: невалидный ответ Gemini → приложение решает по чистым метрикам. Обучение никогда не блокируется из-за модели.

## 2.8. Профили (несколько профилей на устройстве)

Один профиль = одна языковая пара. Несколько профилей — у каждого свои Профиль, Словарь, Свод правил, Прогресс, Аналитика. При запуске с одним профилем — вход сразу; с несколькими — экран выбора. Экспорт — вся база одним файлом.

## 2.9. Импорт своего словаря и режим интенсива

Пользователь вставляет список слов (copy-paste). Приложение разбирает построчно, дозапрашивает у Gemini формы и эмодзи, помечает «приоритетные». Режим интенсива включается автоматически: приоритетные термины отрабатываются в заданиях и Суперквестах; по мере усвоения (80–90%) подмешиваются обычные темы.

# 3. Вход и главное меню (Boot Sequence)

## 3.1. При старте

- Проверяет ключ Gemini (если нет — мастер ключа).
- Читает локальную базу AILEX_StatusDB. Если базы нет — онбординг.

## 3.2. Ветвление

**Один профиль** → вход сразу. **Несколько профилей** → экран выбора.

**Ветвь А — профиль найден:** меню: 1) начать урок, 2) Суперквест, 3) поговорить, 4) Словарь, 5) Свод правил, 6) сменить/создать профиль, 7) настройки.

**Ветвь Б — профиля нет:** онбординг (раздел 4).

# 4. Онбординг и создание базы

## 4.1. Шаг 1. Сбор анкеты

Вопросы по одному: имя, родной язык, целевой язык, возраст, интересы / сфера деятельности.

## 4.2. Шаг 2. Placement Test v2 (версия 4.17)

### Назначение и ограничения предыдущей версии

Предыдущая версия теста (v1, реализована в v31) запрашивала 2 вопроса на уровень (per_level: 2) — итого максимум 10 вопросов на уровни A1–C1. «Не знаю» и явно неверный выбор считались одинаково, что приводило к остановке теста после первой же пары «не знаю + ошибка». Лексические пробелы (незнание конкретного слова) не отделялись от грамматических ошибок. Результат был нестабилен.

Версия 4.17 исправляет эти недостатки тремя независимыми правками.

### Изменение 1. Объём — 3 вопроса на уровень

per_level изменено с 2 на 3. Итого до 15 вопросов на уровни A1–C1. Тест по-прежнему может завершиться досрочно при плохих результатах.

### Изменение 2. Разделение типов вопроса: grammar / vocab

Каждый вопрос получает поле skill: "grammar" | "vocab".

**grammar-вопросы** (2 из 3 на уровень) — проверяют структуру языка: порядок слов, спряжение, артикли, падежи, времена. ОБЯЗАТЕЛЬНОЕ ПРАВИЛО: лексика в таком вопросе берётся из уровня НИЖЕ тестируемого (т.е. вопрос по грамматике A2 использует слова уровня A1). Это гарантирует, что незнакомое слово не маскируется под незнание правила.

**vocab-вопросы** (1 из 3 на уровень) — проверяют узнавание лексики текущего уровня. Участвуют в определении итогового уровня, но НЕ участвуют в счётчике досрочной остановки — незнание слова не останавливает тест.

### Изменение 3. Раздельные счётчики: wrongChoice и dontKnow

Вместо одного счётчика wrongAtLevel[level] вводятся два:

- wrongChoice[level] — пользователь осознанно выбрал неверный вариант ответа
- dontKnow[level] — пользователь нажал «Не знаю»

**Правило досрочной остановки (строже прежнего только для grammar-вопросов):**
Тест останавливается на уровне L, если выполняется хотя бы одно из условий:
- wrongChoice[L] >= 2 (два явных неверных выбора — сильный сигнал незнания)
- wrongChoice[L] + dontKnow[L] >= 3 (сумма ошибок и «не знаю» ≥ 3 из 3 вопросов уровня)

Одно «Не знаю» в сочетании с одним верным ответом — тест продолжается.

### Порог подтверждения уровня — большинство (2 из 3)

Уровень L считается подтверждённым, если passed[L] >= 2 (то есть не менее 2 верных ответов из 3). Прежнее правило «хотя бы 1 из 2» было слишком мягким.

### Маршрутизация по результату

- Passed A1 (2+ верных на A1, тест не остановлен на A1): → A1.1.01 или выше
- Passed['A1'] === 0 (ни одного верного на A1): → A0.1.01 (уровень Pre-A1)
- Ошибка API / нет сети: → A1.1.01 (как было, fallback)

Ответ: { "questions": [ { "level": "A1", "skill": "grammar", "question": "Выберите правильный порядок слов: ___", "options": ["Ich arbeite heute.", "Heute arbeite ich.", "Ich heute arbeite.", "Arbeite heute ich."], "correct_index": 0 }, { "level": "A1", "skill": "vocab", "question": "Что означает слово «der Tisch»?", "options": ["стол","стул","окно","дверь"], "correct_index": 0 } ] }

Требования к grammar-вопросам: лексика строго из уровня НИЖЕ тестируемого; вопрос проверяет структурное правило, а не знание конкретного слова. Требования к vocab-вопросам: слова типичны для тестируемого уровня; все варианты в одной части речи. Ровно по 3 вопроса на уровень, по возрастанию сложности внутри уровня.

### Экран placementIntro — два пути вместо одного

Вместо одной кнопки-заглушки «Я новичок — начать с нуля (A1)» появляются ДВЕ кнопки с явным различием:

- «Не знаю язык совсем — начать с основ (A0)» → finishOnboarding(profile, 'A0')
- «Новичок, но немного знаю — пройти короткий тест» → runPlacement(profile)

Кнопка «Начать тест» остаётся как основной путь для тех, кто уверен в своём уровне.

**Обоснование двух кнопок:** человек лучше приложения знает, знает ли он вообще язык. Самодиагностика здесь надёжнее автоматического теста. Тест по-прежнему может сам направить на A0, если пользователь пошёл по пути теста, но набрал 0 на A1.

## 4.3. Уровень A0 — Pre-A1 (новый раздел, версия 4.17)

### Педагогическое обоснование

По официальной шкале CEFR (Companion Volume 2018, Совет Европы) уровень Pre-A1 зафиксирован отдельно. Он описывает человека, который: понимает отдельные знакомые слова и заученные шаблоны; может задать простейший вопрос из шаблона; не строит предложений свободно; воспринимает грамматику только как «так говорят», а не как систему. В AILEX этот уровень называется A0 и является подготовительным этапом перед A1 — без экзамена и без полноценного цикла 21-задания.

### Кто попадает на A0

- Пользователь нажал кнопку «Не знаю язык совсем» в онбординге
- Placement test вернул passed['A1'] === 0 (ни одного верного на A1)

### Структура контента A0 — 9 тематических блоков

Принцип situated learning («обучение в контексте»): с блока 3 каждый урок завершается мини-сценой — коротким диалогом (3-5 реплик) в реальной бытовой ситуации. Нарратив на родном языке; ключевые фразы встроены на целевом. Задача пользователя — не создавать текст, а выбрать готовую фразу из предложенных вариантов или тапнуть для озвучки. Фонетические блоки (1-2) мини-сцены не получают — на этом этапе человек ещё не воспринимает связный текст.

**Блок 1 — Звуки и чтение: алфавит (1-2 урока)**
Полный алфавит целевого языка с транскрипцией. Отображается тапабельной сеткой (все буквы сразу); тап → TTS. Упражнения: ТОЛЬКО фонетические — буква→звук (6 шт.) и звук→буква (6 шт.), итого 12. Лексические вопросы («что значит слово?») запрещены — человек ещё не знает языка. Режим Gemini: mode="alphabet".

**Блок 2 — Звуки и чтение: буквосочетания (1-2 урока)**
Диграфы и трудные сочетания (sch, ch, ei, ie, ü, ö, ä для немецкого и т.п.). Карточки с примерами и 12 упражнениями. Режим Gemini: mode="alphabet".

**Блок 3 — Первый контакт (1 урок) 🎭 мини-сцена**
Приветствия, прощания, «как вас зовут», «меня зовут», «откуда вы», «я из...». Chunk learning — заученные формулы целиком. Мини-сцена: знакомство с незнакомым человеком.

**Блок 4 — Числа и цвета (1 урок)**
Числа 1-100, основные цвета. Без мини-сцены — лексика тренируется упражнениями на узнавание.

**Блок 5 — Время и даты (1 урок) 🎭 мини-сцена**
Сегодня/завтра/вчера, утро/день/вечер/ночь, дни недели, месяцы. Мини-сцена: спросить время или договориться о встрече.

**Блок 6 — Мой мир (1 урок)**
Члены семьи, 5-6 базовых профессий, предметы вокруг (телефон, ключ, деньги, сумка, дом, машина). Без мини-сцены — лексический урок.

**Блок 7 — Еда, тело и самочувствие (1-2 урока) 🎭 мини-сцена**
Еда и напитки (выживательный минимум: вода, хлеб, кофе, суп, яблоко), части тела (голова, рука, нога, живот, спина), самочувствие (болит, плохо, хорошо). Мини-сцена: аптека или приём у врача.

**Блок 8 — Ориентация и магазин (1-2 урока) 🎭 мини-сцена**
Вопросительные слова (где? что? как? когда? сколько?), направления (направо/налево/прямо), транспорт (базовый), магазин и цены («Сколько стоит?», «Дайте, пожалуйста», «Спасибо»). Мини-сцена: спросить дорогу или купить что-то.

**Блок 9 — Первые предложения (1-2 урока — финал A0) 🎭 мини-сцена**
Самые базовые конструкции: «Я + существительное/прилагательное» (Ich bin Mechaniker), «Это + существительное» (Das ist ein Tisch), «У меня есть» (Ich habe), «Я хочу» (Ich möchte). Мини-сцена: знакомство + короткий рассказ о себе. Мост к A1.

### Объём и масштабирование

Приложение задаёт рамку по дистанции письменности, Gemini наполняет содержание:

| Дистанция | Объём A0 |
| --- | --- |

| Неалфавитная письменность (кит., япон.) | ~14–15 уроков |

### Коды уроков и переход

- Коды: A0.1.01, A0.1.02, … A0.1.NN (подуровень всегда 1)
- Схема «2 подуровня × 25 уроков» применяется только с A1 — A0 вне неё
- После последнего урока A0: автоматический переход на A1.1.01
- Экзамена нет: A0 — подготовительный этап, не CEFR-уровень
- Запись в прогресс: обычная строка A0.1.NN → A0.1.(NN+1) или A1.1.01 для последнего

### Принципы методики A0

- Транскрипция — центральный элемент каждого шага, а не вспомогательный
- TTS — обязательно для каждого звука/слова/шаблона; слушать первично
- Никакого production (свободного написания) до блока 9
- Все задания — узнавание: тап, выбор варианта, сопоставление
- Объяснения строго на родном языке (native_lang), как и везде
- Situated learning (с блока 3): мини-сцена в конце урока показывает изученные фразы в реальной бытовой ситуации; задача — выбор готовой фразы или её озвучка, не свободное построение
- Gemini генерирует контент через контракт a0_lesson (Пример F, раздел 2.5); решение о переходах — за приложением

### Что A0 НЕ включает

- Экзамен (нет)
- Возврат из главного меню (A0 — одноразовый онбординговый путь)
- Суперквест на уровне A0 (нет — слишком мало материала)
- Полный цикл 21-задания (формат другой — короче и легче)

# 5. Методология урока (The 21-Drill System)

Применяется для уровней A1 и выше. Уровень A0 использует собственный более простой формат (раздел 4.3); после перехода на A1 — полноценный 21-drill.

## 5.0. Глобальные директивы

- **Соответствие CEFR:** весь контент строго соответствует текущему уровню урока.
- **Родной контекст:** вся теория, объяснения и подсказки — строго на родном языке, с мнемотехниками.
- **No back-translation:** в упражнениях запрещено просить перевод с целевого языка на родной.

### Чтение и звучание (транскрипция + озвучка)

- Обязательная транскрипция родными буквами для каждого нового слова (например [хэлоу]).
- Трудные слова повторяются с транскрипцией при каждом повторе.
- Озвучка (TTS) доступна для слов и фраз — кнопка прослушивания.
- Честная оговорка: транскрипция приблизительна. Это опора для чтения, а не эталон.

### Движок квестов и рассказов (Quest Core)

Атмосфера строится на интересах пользователя из профиля (не фиксированная НФ-стилистика). Anti-Frustration: после 2 ошибок AILEX даёт правильный ответ через мнемотехнику и завершает квест.

### Генератор лексики (Vocabulary Engine)

- Фильтр новизны: дубликаты из KNOWN_VOCAB исключаются.
- 7 единиц: если есть неправильные глаголы — 5 новых слов + 2 неправильных; иначе — 7 новых.

### Баланс лексики: интересы + бытовое

A1 — 40% бытовое / 60% интересы; A2 — 35/65; B1 — 30/70; B2 — 25/75; C1–C2 — 20/80.

На A0 — 100% выживательная/бытовая (весь курс подготовительный).

### Структура 21 задания

- Повторение (7): повторение из предыдущего урока (термин «Recap» устарел, заменён на «Повторение» в v34)
- Presentation: таблица новых слов
- Grammar Card: объяснение нового правила (показывается только если есть новое правило)
- Задания 1–5 (Recognition): выбор варианта
- Задания 6–11 (Writing): написание по памяти
- Задания 12–20 (Construction): построение фраз и предложений
- Задание 21 (Final Challenge): интерактивный рассказ-квест (раздел 5.A)

**Проверка по смыслу:** равноценные формы (im = in dem, zum, am) засчитываются верными. Опечатка 1–2 символа — верно с пометкой. Ошибка только в артикле — «почти верно», не блокирует, копится в адаптивное закрепление.

**Подсказки-скелеты:** скрывают именно целевой элемент задания (артикль/спряжение/порядок/слово).

**Полная подсказка (4.14):** правильный ответ целиком с транскрипцией, переводом и озвучкой.

## 5.A. Интерактивный рассказ-квест с развилками

Гибридный движок: жёсткий каркас (приложение) + живая реакция (Gemini). Структура: 2 развилки по 2 варианта каждая. Масштабирование по CEFR: A1 — 4–6 предложений; A2 — 7–10; B1 — 12–15; B2 — 16–20; C1–C2 — 20+.

## 5.B. Голосовой диалог (опция «Поговорить»)

Цепочка STT → Gemini → TTS. Двойной ввод (голос/текст). Роль «напарник, а не судья». Метрика фиксируется в аналитике.

## 5.1. Протокол сохранения

Автозапись после последнего задания: строка в «Прогресс», новые слова в «Словарь», метрики в «Аналитику». Уведомление «Сохранено ✓».

# 6. Суперквест

**Статус:** реализован в коде (билд v31, GitHub Pages 4wood70/ailex).

Единый квест-погружение на 30–50 предложений, разбитый на акты. Повторение лексики происходит внутри живой истории.

## 6.1. Модель движка

Наследуется от движка квеста (раздел 5.A): гибрид каркас+реакция; Anti-Frustration/напарник; запись в Словарь.

## 6.2. Источники и баланс лексики

80% — повторение KNOWN_VOCAB; 20% — новая функциональная лексика. Баланс бытовое/интересы — по CEFR (как в Vocabulary Engine).

## 6.3. Структура повествования: акты

| Уровень | Число актов | Общий объём |
| --- | --- | --- |
| A1 | 3 акта | ~30 предложений |
| A2 | 3 акта | ~34 предложения |
| B1 | 4 акта | ~40 предложений |
| B2 | 4 акта | ~44 предложения |
| C1–C2 | 5 актов | ~50 предложений |

Между актами — экран-итог («Глава N пройдена»). Последний акт завершает историю.

## 6.4. Развилки

Число не фиксируется жёстко. Граница: не более 1–2 на акт. Где ставить — решает Gemini по логике повествования.

## 6.5. Тап-перевод новых слов

Слова из encountered_words кликабельны в тексте. Тап → карточка (перевод + транскрипция + озвучка). Дополнение к списку 📒 «Слова задания», не замена.

## 6.6. Anti-Frustration на длинной дистанции

Счётчик напарника обнуляется в начале каждого акта. Счётчик итоговой точности копится за весь Суперквест без сброса.

## 6.7. Производительность и подгрузка

Следующий акт начинает готовиться в фоне, пока пользователь проходит текущий.

## 6.8. Завершение и сохранение

Запись: SUPER_[код]. Итоговая точность — по накопленным за весь Суперквест данным.

## 6.9. Контракт с Gemini

Схема quest_step (раздел 2.5). Дополнительные поля: act_index/acts_total, branches_used_in_act/branch_limit_in_act (граница 2), vocab_balance (80/20), topic_mode. Ответ: is_act_final (текущая сцена завершает акт) и is_final (завершает всю историю) — два независимых булевых поля (реализовано в v31).

# 7. Экзамены и завершение уровня

График: Checkpoint после 25 уроков, Final Exam после 50 уроков. Тест — 50 вопросов. Порог сдачи — 85%.

- Успех (≥85%): EXAM_[Уровень]_PASS, переход на следующий уровень.
- Первый провал: EXAM_[Уровень]_FAIL-1, возврат к экзамену.
- Второй провал: переход в режим работы над ошибками (раздел 8).

**Уровень A0 — без экзамена** (см. раздел 4.3).

# 8. Режим работы над ошибками (Remedial)

Активируется после второго провала экзамена.

- Разбор: объяснение ошибок на родном языке с мнемотехниками.
- Тренинг: по 2 штрафных задания на каждую ошибку.
- После тренинга: запись EXAM_[Уровень]_REMEDIAL → возврат к экзамену.

# 9. Визуализация и мнемоника

## 9.1. Графический стандарт

Маркеры: 💡 логика, 🎯 задание, 🌍 культура, 🧠 запомни, ✅ / ❌, 💾 сохранение. Транскрипция обязательна для каждого нового слова. TTS доступна для слов и фраз при наличии поддержки браузером; при отсутствии озвучки — только транскрипция. Не более одного задания на экран.

## 9.2. Универсальные визуальные мнемоники

Формат: [слово] → [эмодзи] → [форма / перевод]. Примеры: trinken → 🥤 → getrunken = пить; sehen → 👀 → gesehen = видеть.

- Версия 4.20 (механика заданий с пропуском, реализуется в v36, 23.06.2026).

  (A) Механика ввода в заданиях с пропуском. Затронуты два типа заданий: «Повторение в контексте» (drillReviewContext, фаза Повторение, R1/7) и «Слово в контексте» (drillContext, фаза Слово в контексте, Задание 6–11).

  **Было:** поле ввода принимает одно слово; формулировка «Вспомните слово и впишите его в пропуск» / «Дополните предложение: впишите ОДНО пропущенное слово»; placeholder «Одно слово».

  **Стало:** поле ввода принимает полное предложение; формулировка «Напишите предложение, подставив пропущенное слово»; placeholder «Напишите предложение целиком». После верного ответа отображается полное предложение с кнопкой 🔊 (при наличии TTS).

  **Логика проверки — два равноценных пути к «верно»:**
  1. Ввод содержит целое предложение → из ввода извлекается слово на позиции пропуска (по токенам) и сравнивается с `answer` (Левенштейн ≤1).
  2. Ввод совпадает только с одним словом `answer` (совместимость со старой привычкой) → засчитывается, но в обратной связи показывается полное правильное предложение.

  Путь 2 — мягкая деградация: не ломает пользователя, который по привычке вводит одно слово, и одновременно стимулирует перейти к полному вводу.

  **Отвергнутые варианты:**
  - «Оставить одно слово» — формулировка «впишите в пропуск» вводила в заблуждение: задание показывает предложение, а поле ждёт только слово, что создаёт несоответствие UX.
  - «Полная проверка предложения через Gemini» — избыточная нагрузка на API; суть задания — отработка одного слова, а не всего предложения.
  - «Принимать только полное предложение, отклонять одно слово» — ломает пользователей, привыкших к старой механике; не дружелюбно при вводе с мобильной клавиатуры.
