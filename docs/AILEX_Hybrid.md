# О документе

Данный документ фиксирует архитектуру гибридной версии AILEX и её отличия от базовой (AILEX 4.4). Все методологические решения — методика 21-drill, контракты Gemini, движок квестов, структура базы данных — наследуются из AILEX 4.4 без изменений. Новизна гибридной версии касается исключительно модели доступа к API и монетизации.

Ключевой принцип AILEX 4.4, сохранённый в гибридной версии: логика → приложение, контент → Gemini, хранилище → устройство пользователя (локально). Смена версии Gemini не меняет методику.

# 1. Текущее состояние — AILEX 4.4

## 1.1 Реализованный функционал

- Android APK — собран через PWABuilder (TWA), установлен и работает на устройстве

- HTML-приложение (PWA) опубликовано на GitHub Pages — служит источником для TWA

- Онбординг с мастером получения Gemini API ключа — кнопка открывает Google AI Studio

- Методология 21-drill — полный цикл урока

- Адаптивный placement-тест (18 вопросов, досрочная остановка при ошибках)

- Нарративные квесты с развилками (2 ветки × 2 выбора, движок 5.A из AILEX 4.4)

- Словарь (лист «Словарь» — по строке на слово с мнемониками тир 1: текст + эмодзи)

- Аналитика / прогресс (экран Analytics)

- Голосовой диалог: опция «Поговорить» STT→Gemini→TTS (раздел 5.B AILEX 4.4)

- Движок адаптации «Gemini советует — приложение решает» (раздел 2.7 AILEX 4.4)

- База данных — локально на устройстве (IndexedDB / localStorage внутри APK); прогресс сохраняется автоматически на смартфоне пользователя

- Экспорт/импорт базы через JSON-файл (ручная синхронизация между устройствами)

- Выбор моделей Gemini с автоматическим перебором: основная gemini-3.5-flash, далее gemini-flash-latest, gemini-2.5-flash, gemini-2.0-flash. Список и имена моделей вынесены в одно место кода и обновляются под актуальный набор аккаунта

- Устойчивость к сбоям модели: при пустом или некорректном ответе (в т. ч. от “думающих” моделей 3.x) приложение молча пробует следующую модель; для генерации задан thinkingBudget = 0, чтобы модель сразу отдавала JSON урока. Экран ошибки урока показывает причину сбоя

## 1.3 Техническая архитектура базовой версии

- Frontend: один файл index.html (HTML + CSS + JavaScript)

- Хостинг: GitHub Pages → github.io/ailex/

- Android: APK собран и работает (TWA через PWABuilder); устанавливается на смартфон

- ИИ: прямые запросы к Gemini API с устройства пользователя; модель gemini-3.5-flash (с перебором запасных), thinkingBudget = 0, строгий JSON по контрактам раздела 2.5 AILEX 4.4

- Хранилище: IndexedDB / localStorage на устройстве — прогресс хранится локально на смартфоне

## 1.4 Технология реализации гибридной версии (нативная, Flutter)

Если основная версия (AILEX 4.4) собрана веб-средствами (HTML/CSS/JavaScript, PWA, APK через PWABuilder — см. парный документ), то гибридная (нативная) версия реализуется на Flutter — фреймворке Google, где один код на языке Dart собирается в нативные приложения сразу под Android и потенциально под iPhone. Методика и логика обучения переносятся без изменений по спецификации AILEX 4.4; меняется только форма исполнения. Преимущества нативной формы: более плавная работа и запуск, независимость интерфейса от интернета (сеть нужна только Gemini), штатная поддержка рекламы (AdMob) и push-уведомлений (см. раздел 7.3). Прогресс хранится локально на устройстве, ключ Gemini — у пользователя, как и в основной версии.

**Инструменты и где их взять: **Flutter SDK (язык Dart) — [https://flutter.dev](https://flutter.dev) , инструкция по установке — [https://docs.flutter.dev/get-started/install](https://docs.flutter.dev/get-started/install) . Среда разработки и сборка APK/AAB — Android Studio: [https://developer.android.com/studio](https://developer.android.com/studio) . Статус: нативная версия — будущий этап; на момент 4.4 рабочим продуктом остаётся основная веб-версия.

Риск: приложение зависит от доступности GitHub Pages. При удалении репозитория или сбое хостинга приложение перестаёт работать у всех пользователей.

# 2. Гибридная версия — архитектура и отличия

## 2.1 Ключевая идея

Гибридная модель устраняет главный барьер входа: пользователю не нужно получать API ключ при первом запуске. Первые 10 уроков генерируются через прокси-сервер разработчика (за счёт разработчика). После исчерпания бесплатного лимита пользователю предлагается выбор: ввести свой Gemini API ключ (бесплатно, через мастер из AILEX 4.4) или подключить платную подписку.

## 2.2 Схема запроса

**AILEX (телефон)  →  Прокси-сервер (Cloudflare Workers)  →  Gemini API (Google)**

- В бесплатном режиме: запросы идут через прокси, который подставляет ключ разработчика

- Прокси считает израсходованные уроки в Cloudflare KV (ключи вида lessons:userId)

- Если пользователь ввёл свой ключ: запросы идут напрямую к Gemini, минуя прокси

- Gemini API ключ пользователя хранится локально, разработчик его не видит

## 2.3 Прокси-сервер (worker.js)

- Платформа: Cloudflare Workers (бесплатный план — 100 000 запросов/день)

- Язык: JavaScript ES Modules (файл worker.js готов)

- Маршруты:

- POST /generate — генерация контента через Gemini API

- GET /check?userId=X — проверка остатка бесплатных уроков

- Хранилище счётчика: Cloudflare KV Namespace (AILEX_KV)

- Лимит: 10 бесплатных уроков на одного пользователя

- CORS: разрешён для всех доменов (GitHub Pages + будущий домен)

- Файлы: worker.js + wrangler.toml (готовы к деплою)

## 2.4 Изменения в приложении (index.html)

- Заменить прямой URL Gemini API на URL прокси — одна строка кода

- Добавить счётчик бесплатных уроков на экране прогресса

- Добавить экран-предложение после исчерпания лимита (ввести ключ / подписка)

- Добавить splash screen (заставка при запуске — логотип AILEX, 3 сек)

## 2.5 Что НЕ меняется в гибридной версии

Вся методология AILEX 4.4 сохраняется полностью: 21-drill, контракты Gemini (JSON), движок квестов 5.A, голосовой диалог 5.B, движок адаптации 2.7, структура словаря, аналитика, онбординг, placement-тест.

- Будущая нативная версия (Kotlin/Flutter): Google AdMob — полноценная мобильная реклама

# 7. Будущие этапы (дорожная карта)

## 7.1 Хранилище и синхронизация

- Google Sheets (AILEX_StatusDB): OAuth + drive.file → полноценная облачная БД из AILEX 4.4

- 4 листа: «Профиль», «Прогресс», «Словарь», «Аналитика» — по спецификации раздела 2 AILEX 4.4

- Синхронизация прогресса при смене устройства — продолжение с того же места

## 7.2 Мнемоники тир 2 и 3

- Тир 2: схемы из данных (таблицы спряжения, цепочки со стрелками) — рисуются на лету

- Тир 3: Imagen — AI-картинка для стойко-трудных слов, хранится на Drive пользователя

## 7.3 Нативная Android-версия

- Kotlin или Flutter — полноценное нативное приложение

- Google AdMob (значительно выше доход от рекламы)

- Push-уведомления (напоминания об уроках)

- Оценка произношения в голосовом диалоге

## 7.4 Контент

- Языки: обучение (пара “родной → целевой”) доступно практически для любого из 100+ языков, поддерживаемых Gemini — контент, переводы и транскрипция генерируются моделью. Расширяется лишь готовый перевод кнопок/надписей интерфейса (приоритет: французский, испанский, итальянский, португальский); до перевода кнопки показываются на английском, обучение всё равно идёт на родном языке.

- Специализированные треки (бизнес, путешествия, IT)
