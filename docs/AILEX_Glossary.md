# 1. Принцип терминологии

Методические термины в интерфейсе — на родном языке учащегося, идиоматично. Не транслитерация английского, а естественное слово, понятное носителю (для русского «задание», для немца — своё немецкое слово и т.д.). По смыслу одно и то же на всех языках.

Английские названия в коде (имена функций, служебные ключи к Gemini, коды состояний) — внутренние, пользователю не показываются и не меняются.

Механика: исходные строки интерфейса — русские; для других языков их переводит слой локализации (встроенный словарь + автоперевод Gemini с кэшем). Поэтому достаточно держать русские строки чистыми.

# 3. Ключевые понятия методики

**Цикл урока (21 задание) — **полный урок состоит из 21 задания: повторение, презентация новых слов, грамблок, отработка, квест.

**Профиль — **один профиль = одна языковая пара «родной → целевой». На устройстве может быть несколько профилей; данные не смешиваются.

**Баланс «бытовое / интересы» — **доля тем по уровню CEFR. A1 40/60, A2 35/65, B1 30/70, B2 25/75, C1–C2 20/80 (бытовое/интересы). Бытовое: магазин, транспорт, деньги, еда, здоровье, вежливость, время. Интересы: из профиля.

**Баланс супер-урока / Суперквеста — **80% повторение известной лексики, 20% новые служебные слова (предлоги, союзы, частицы).

**Движок адаптации — **«Gemini советует — приложение решает»: модель даёт диагноз ошибок в строгих категориях, действие выбирает приложение. Диагноз гибкий — лечение жёсткое.

**Тиры мнемоник — **тир 1 — текст + эмодзи (реализовано); тир 2 — схемы из данных; тир 3 — AI-картинки (Imagen). Тиры 2–3 — будущее, с откатом тир 3 → 2 → 1.

**Прогресс-индикатор — **кольцо на главном экране — продвижение по уровню (по пройденным урокам, 50 = 100%); отдельная подпись — точность (accuracy_first_try). Два разных показателя, не путать.

# 4. Принятые архитектурные решения (закрыто, не переоткрывать)

- Разделение: логика → приложение (state machine), контент → Gemini, хранилище → локальная база на устройстве.

- Смена версии модели Gemini НЕ меняет методику обучения.

- Gemini — только генератор контента, всегда строго в JSON по контракту; о базе не знает, методику не определяет.

- Хранилище сейчас — локальная база + экспорт/импорт JSON. Google Sheets / Drive (облако, OAuth) — будущий этап, не текущий.

- Гибридная версия (прокси Cloudflare, freemium, подписка, Google Play) — заготовка «на полке». Решение о платной модели НЕ принято; будет после собственного тестирования и фокус-группы.

- Версионирование: APP_VERSION в index.html и CACHE в sw.js всегда совпадают и поднимаются при каждой правке (иначе залипает кэш PWA). Проставляет Claude.

- Рабочий принцип изменений: сначала спецификация → согласование → потом код. Методику спонтанными правками не трогаем.
