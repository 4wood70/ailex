# О документе

Это «конституция» методики AILEX — описание логики обучения, которая закладывается в код приложения жёстко и не зависит от версии используемой модели Gemini. Gemini выступает только генератором контента (слова, фразы, проверка ответов, интерактивные рассказы-квесты), а вся последовательность и правила обучения принадлежат приложению.

**Ключевой принцип разделения ответственности:** логика → приложение, контент → Gemini, хранилище → локальная база на устройстве (облако на Google Drive — будущий этап). Смена версии Gemini не должна менять методику.

## Архитектура (общая картина)

- Платформа: Android-приложение (публикация в Google Play). Устройство поддерживает несколько профилей (раздел 2.8): один профиль = одна языковая пара «родной → целевой». Это позволяет учить несколько языков (по профилю на язык) и использовать приложение нескольким людям на одном устройстве. У каждого профиля свои словарь, свод правил, прогресс и аналитика — полностью раздельно.

- Логика обучения: конечный автомат (state machine) внутри приложения — профиль, placement-тест, уроки, чекпоинт/экзамен, режим работы над ошибками.

- Контент: запросы к Gemini API с жёсткими рамками (уровень CEFR, обязательная лексика, длина, запрет оффтопа).

- Хранилище (реализованный билд): локальная база на устройстве AILEX_StatusDB (структура из пяти частей — «Профиль», «Прогресс», «Словарь», «Свод правил» и «Аналитика»); перенос между устройствами — вручную экспортом/импортом JSON. Без входа в Google и без OAuth. Облачный вариант (Google Sheets на Drive) — цель будущего этапа.

- Ключ Gemini: вводится самим пользователем через мастер получения ключа (раздел 0); каждый пользователь использует и оплачивает свой ключ.

## Технология реализации (как собрана основная версия)

Основная (текущая) версия AILEX реализована веб-средствами: HTML, CSS и JavaScript, собрана одним самостоятельным файлом index.html. Файл оформлен как PWA: добавлены manifest.webmanifest, сервис-воркер sw.js и иконки. Хранение прогресса — локальное на устройстве; запросы к Gemini идут напрямую с устройства по ключу пользователя.

**Где лежат исходники:** репозиторий https://github.com/4wood70/ailex, опубликован на GitHub Pages. APK собирается через PWABuilder (TWA). Обновление кода: заменить файлы в репозитории — пересборка APK не нужна.

# 0. Техническая подготовка и доступ

- При первом запуске вход в Google-аккаунт не требуется. Прогресс хранится локально на устройстве (раздел 2.0).

- **Автосохранение без участия пользователя:** запись прогресса выполняется автоматически по завершении этапа.

## Мастер получения ключа Gemini

Цель: превратить «найди и скопируй API-ключ» в простой пошаговый помощник. Делается один раз; дальше ключ хранится в приложении.

- Ключ создаётся в один-два тапа. Приложение предлагает вставить его в поле.
- Ключ сохраняется в защищённом хранилище устройства.

# 1. Академический фокус (Topic Lockdown)

- **Роль:** AILEX — учитель иностранных языков по стандарту CEFR. Стиль: строгий, лаконичный, дисциплинированный наставник.

- **Жёсткий фокус:** обсуждаются только темы обучения. На оффтоп — Pivot-метод.

- **Язык общения:** все объяснения, комментарии и служебные тексты — строго на родном языке пользователя.

- **Управление состоянием обучения принадлежит приложению**, а не модели.

## Язык интерфейса и пояснений (редакция 4.25)

**Ключевой принцип:** язык интерфейса привязан к **родному языку профиля**, а не к языку устройства. Всё, что пользователь должен понимать смыслом — кнопки, пункты меню, заголовки, подсказки, справка, разборы ошибок, правила — на родном языке того, кто занимается по этому профилю.

### Откуда берётся язык интерфейса

- **Самый первый вход, ни одного профиля ещё нет.** Интерфейс — на системном языке устройства. Причина: человек ещё не известен, надо лишь довести его до создания профиля и выбора родного языка.
- **Профиль создан.** Язык интерфейса = родной язык, выбранный в этом профиле.
- **Несколько профилей.** У каждого профиля свой родной язык; при переключении профилей интерфейс переключается вместе с ними. Если второй профиль завёл другой человек с другим родным языком — для него интерфейс будет на его языке.
- **Отдельной опции «язык интерфейса» в Настройках нет.** Родной язык задаётся при создании профиля; сменить его можно через создание/удаление профиля (эта возможность уже есть). Обоснование отказа от ручной опции: тем же Gemini на родной язык переводится и весь учебный контент — если для редкого языка перевод выйдет неудачным, ручной переключатель интерфейса не помог бы, учебный материал остался бы на том же языке.

### Механизм перевода — два пути

Тексты интерфейса делятся на два вида; для каждого свой путь. Оба кэшируются на устройстве и работают офлайн после первой загрузки. Русский — исходный язык всех строк.

- **Короткие надписи** (кнопки, пункты меню, заголовки, плейсхолдеры). Путь существует с 4.18 — контракт `translate_ui`: каждая строка идёт в Gemini с описанием её функции в квадратных скобках, правило требует подобрать **идиоматичное** слово целевого языка, а не дословный перевод. Пример смысла: «артикль» есть в немецком, отсутствует в русском, а в китайском/японском выражается иначе — Gemini подбирает естественный термин или описательную форму под конкретный язык. В 4.25 достраивается **охват**: все видимые надписи проводятся через реестр (Приложение А), а не только те, что случайно прошли через функцию `t()`.
- **Целые абзацы** (подсказки экранов `HintTexts`, справка «Как пользоваться»). Новый путь. Это абзацы с выделением и эмодзи, механизм точного совпадения из первого пути их не обрабатывает. Каждый блок переводится как единый текст с сохранением выделения и эмодзи, результат кэшируется по языку.

### Момент загрузки и порядок деградации

- **Когда грузятся переводы.** Один раз, при создании профиля (родной язык уже выбран, ключ Gemini уже введён). К первому показу экранов всё готово; далее — из сохранённого хранилища, в том числе офлайн. Это и есть основной механизм офлайна для **любого** языка.
- **Английская заготовка** (встроенный словарь `en`) сохраняется дополнительно — как мгновенный запас без единого запроса для случая, когда родной язык английский, а интернета при самом первом входе нет.
- **Порядок деградации** (если готового перевода нет): сохранённый перевод на устройстве → запрос к Gemini → английская заготовка (только для английского) → русский исходник. Интерфейс и обучение никогда не блокируются отсутствием перевода.

### Учебные пояснения от Gemini

Разбор ошибок, правила, объяснения «почему так» — как и прежде, генерируются на родном языке через параметр `native_lang` контракта (независимо от механизма локализации надписей).

## Автозавершение этапа (Auto-Save Control)

Явное согласие на сохранение убрано. При достижении финала этапа приложение немедленно и автоматически сохраняет прогресс. Показывается уведомление «Сохранено ✓», которое само исчезает.

# 2. Архитектура данных и памяти

## 2.0. Хранилище данных

**Статус реализации.** Данные хранятся локально на устройстве — база AILEX_StatusDB, без входа в Google и без OAuth. Перенос — вручную через экспорт/импорт файла AILEX_StatusDB.json. Облачный вариант (Google Sheets на Drive) — будущий этап.

## 2.1. Логика индексации

- Индекс урока: [Уровень].[Подуровень].[Номер], например A1.1.01; B2.2.05.
- Уровень A0 (Pre-A1): коды A0.1.01…A0.1.NN (подуровень всегда 1, экзамена нет — раздел 4.3).

## 2.2. Лист «Профиль» — статика

Состав (6 полей): Имя, Родной язык, Целевой язык, Возраст, Интересы / сфера деятельности, Уровень CEFR (определяется placement-тестом; может быть A0).

## 2.3. Лист «Прогресс» — динамика

Назначение: история обучения — лента событий. Рекомендуемые столбцы: Код этапа, Дата, Слова этапа, Следующий этап, Примечание.

### Сценарии записи

- Стандартный урок пройден: A1.1.05 | 01.03.26 | слово1, слово2 | A1.1.06
- Урок A0 пройден: A0.1.03 | 22.06.26 | (фонема) | A0.1.04
- Завершение A0: A0.1.NN | 22.06.26 | (пусто) | A1.1.01
- Суперквест пройден: SUPER_A1.1.05 | 01.03.26 | предлог1 | A1.1.06
- Экзамен сдан: EXAM_A1.1_PASS | 01.03.26 | (пусто) | A1.2.01
- Экзамен провален (1-я попытка): EXAM_A1.1_FAIL-1 | 01.03.26 | (пусто) | EXAM_A1.1
- REMEDIAL: EXAM_A1.1_REMEDIAL | 01.03.26 | ошибка1 | EXAM_A1.1

## 2.3-С. Лист «Словарь»

Полноценный личный словарь — по одной строке на слово (KNOWN_VOCAB).

Поля: Слово (целевой язык), Перевод (родной), Транскрипция (родными буквами), Грамматические формы («паспорт»), Эмодзи-якорь, Неправильный глагол (да/нет), Урок-источник, Число ошибок, Статус освоения, Мнемоника, Даты.

**Прогресс-индикатор:** Большое кольцо — прогресс по ТЕКУЩЕМУ уровню CEFR (по пройденным урокам: 50 уроков на уровень A1+; для A0 — своя шкала, 100% после последнего урока A0). Мелкая подпись — точность ответов (accuracy_first_try). Кольцо A0 обнуляется при переходе на A1.

**Каталог словаря по буквам:** индекс по первой букве самого слова без ведущего артикля (der Bus → «B»). Фильтр «только неправильные глаголы».

## 2.3-Я. Лист «Ядро языка» (раздел 4.26)

**Назначение.** Локальный склад слов целевого языка, из которого приложение само набирает порцию урока. Это не словарь пользователя: ядро — сырьё, Словарь — то, что человек уже видел в уроке. Слово переезжает из Ядра в Словарь в момент выдачи и никогда не наоборот.

**Почему отдельный лист.** Словарь отвечает на вопрос «что я выучил» и показывается человеку. Ядро — служебный склад, человеку списком не показывается никогда (см. запрет таблицы ниже). Смешать их — значит обесценить прогресс-индикатор: 200 невыданных глаголов сделают счётчик «слов в словаре» бессмысленным.

**Поля записи:** Слово (целевой язык), Перевод, Транскрипция (родными буквами), Эмодзи-якорь, Часть речи, Артикль рода, Грамматические формы («паспорт»), Неправильный глагол (да/нет), Уровень CEFR, Тема, Класс (закрытый / открытый), Слой (общий / интересы), Статус склада (склад / выдано / архив).

**Два класса слов — разная стратегия загрузки:**

- **Закрытый класс** — местоимения, предлоги, союзы, частицы, артикли, неправильные глаголы. Список в языке конечен и не пополняется (нем.: ~200 неправильных глаголов, ~25 предлогов, ~20 союзов; в других языках объём свой). Грузится **целиком при создании профиля**, независимо от уровня.
- **Открытый класс** — существительные, прилагательные, обычные глаголы, наречия. Пополняется бесконечно, грузить целиком бессмысленно. Грузится **по уровням**: уровень онбординга при создании профиля, следующий — фоном, пока учится текущий.

**Два слоя внутри открытого класса:**

- **Общий слой** — бытовая и выживательная лексика. От интересов не зависит, при смене профессии не трогается.
- **Тематический слой** — 100–200 слов под интересы профиля. Единственный слой, который перегенерируется при смене интересов.

**Загрузить ≠ выдать (ключевое разграничение).** Все закрытые классы лежат в базе с первого дня — приложению они нужны, чтобы узнавать формы в сценах, строить порцию и не дозапрашивать Gemini. Но в порцию урока слово выходит только по методике (раздел 5). Наличие слова в Ядре не даёт человеку ни одного лишнего слова на экране.

**Запрет таблицы (принято 17.07.26).** Ядро не показывается человеку списком ни в каком виде — в частности, отвергнута идея экрана «таблица неправильных глаголов» (все формы разом, как в старых учебниках). Обоснование: таблица целиком не заучивается, это формальная зубрёжка без опоры на контекст. Вместо этого — «голограмма»: глагол выдан в порции урока → всплыл в квесте → тап дал перевод → 📘 объяснил форму → попал в Суперквест через `must_use`. Одно и то же слово в разных ситуациях, а не список. Существующий фильтр Словаря «только неправильные глаголы» сохраняется — он показывает то, что человеку уже выдавали, и таблицей ядра не становится.

**Статус «архив» и смена интересов.** При смене интересов в профиле невыданные слова старой темы получают статус «архив»: они выбывают из набора порции, но остаются в базе. Возврат прежней темы возвращает их в «склад» мгновенно, без повторной генерации. Архив — не запрет: если архивное слово всплыло в сцене квеста, тап-перевод и 📘 «почему так» работают как обычно (эти пути идут по своей логике и Ядра не касаются). Уже выданные слова старой темы остаются в Словаре навсегда — они выучены, актуальность темы к этому отношения не имеет.

**Исчерпание.** Когда в Ядре не осталось слов «склада» подходящего уровня и слоя — приложение откатывается на резервный путь: запрос `generate_vocab` к Gemini по прежней схеме (раздел 5). Обучение не блокируется.

## 2.3-М. Мнемотехники

Генерируются один раз по триггеру для трудных слов/правил/фраз, сохраняются в словарь. Три уровня деградации: (1) текст + эмодзи (реализовано), (2) схемы из данных, (3) Imagen-картинка (будущие этапы).

## 2.3-П. Часть «Свод правил»

Личный справочник грамматики. Каждое правило, показанное как Grammar Card, сохраняется со статусом «изучено». Доступен с главного экрана без запуска урока. Группировка по категориям: Артикли, Существительные, Глаголы, Времена, Падежи, Предлоги, Порядок слов, Местоимения, Прочее.

## 2.4. Разделение ответственности: кто пишет в базу

**Железное правило:** Gemini не знает о существовании базы и никогда не пишет в неё. Всю работу с базой выполняет исключительно приложение.

## 2.5. Контракт вызова Gemini (строгий JSON)

Ответ Gemini — всегда строго структурированный JSON, без свободного текста вне полей.

### Пример A. Генерация лексики урока (с 4.26 — резервный путь)

**Статус с редакции 4.26:** этот контракт больше НЕ является основным способом набрать порцию урока. Порцию собирает приложение из Ядра языка (раздел 2.3-Я, правила набора — раздел 5). `generate_vocab` вызывается только при исчерпании Ядра или сбое его генерации. Причина замены: подбор слов — это методика, а методика по конституции принадлежит приложению; отдав подбор модели, мы получали недобор порции (Gemini предлагал уже известное, фильтр дубликатов резал порцию до 3–6 слов, добора не было).


Запрос: { "task": "generate_vocab", "target_lang": "de", "native_lang": "ru", "cefr": "A2", "topic": "столярное дело", "count": 7, "known_vocab": ["arbeiten", "Holz"] }

Ответ: { "words": [ { "term": "saegen", "emoji": "🪚", "translit": "[зэ:ген]", "translation": "пилить", "part_of_speech": "verb", "is_irregular": false, "forms": "saegt, saegte, hat gesaegt" }, { "term": "Tisch", "emoji": "🪑", "translit": "[тиш]", "translation": "стол", "part_of_speech": "noun", "article": "der", "forms": "der Tisch, die Tische" } ] }

### Пример B. Проверка ответа пользователя

Запрос: { "task": "validate_quest_scene", "target_lang": "de", "native_lang": "ru", "cefr": "A1", "scene_text": "…", "prompt_for_user": "…", "expected_target": "…", "scaffold": "Ich ___ ___.", "structural_collision": false }

**Двухслойная проверка:**
- СТРУКТУРНАЯ (код, до запроса): приложение проверяет, не встречается ли стем каркаса в scene_text уже достроенным иным словом (эталон целиком в сцене отсутствует) → `structural_collision`. Детерминированно ловит праймящую коллизию (QUEUE-36/42).
- СЕМАНТИЧЕСКАЯ (Gemini, этот контракт): четыре критерия — (1) смысл: expected_target отвечает на prompt_for_user на том же уровне вопроса; (2) коллизия: каркас не показан в сцене достроенным иначе; (3) узуальность и корректность слов: эталон — живая фраза без битых/несуществующих слов (например «weiterlauf» не существует) и калек, иначе упростить под уровень CEFR; (4) согласованность глоссария: answer_glossary и форма в expected_target совпадают, при задании-выборе вариантов в глоссарии ≥2 варианта (иначе выбор фиктивный).

Стоимость: +1 запрос Gemini на сцену (по выбору пользователя — семантику проверяет Gemini, локально это не сделать); на практике в RPM-потолок не упирается (узкое место — всплески, не штатная игра по сцене раз в 30–60 с).

## 2.7. Движок адаптации: «Gemini советует — приложение решает»

Трёхуровневая схема: (1) сбор метрик, (2) диагностика Gemini (строгий JSON из словаря категорий), (3) действие по жёстким правилам приложения.

Словарь диагнозов: dominant_error_type (recognition / spelling / verb_conjugation / word_order / vocabulary_gap / none), risk_words, confidence_trend (improving / stable / declining), note_native.

Пример правил: accuracy < 0.70 → вперёд + усиленное повторение; dominant_error_type = verb_conjugation → добавить блок на спряжение; слово с 2+ ошибками → Повторение следующего урока.

**Вход и выход из очереди повторения (уточнено в 4.26, скорректировано в 4.27).** Вход: 2 ошибки на слове → очередь повторения (срочный слой) + генерация мнемоники; трудные слова урока и `risk_words` диагностики — туда же; при точности урока ниже 0.70 в очередь уходит вся порция. **Выход:** слово покидает очередь после **двух уроков подряд, в которых оно встретилось и не дало ни одной ошибки**. **Важно (4.27):** уход из очереди — это возврат слова на спокойную интервальную лестницу (раздел 2.7-И), а НЕ статус «освоено». Прежнее правило «выход из очереди → освоено» и старое авто-«освоено» (0 ошибок за один урок) **отменены**: единственный вход в «освоено» теперь — вершина лестницы 1/3/7/30. Обоснование: в рамках одного урока слово держит оперативная память, это не признак долговременного запоминания.

Защита от сбоя: невалидный ответ Gemini → приложение решает по чистым метрикам. Обучение никогда не блокируется из-за модели.

## 2.7-И. Интервальное повторение слов (спека 4.27, код v65)

Чистая логика приложения, Gemini не задействован: расписание считает state machine по датам. Для показа слова в контексте используется существующий контракт `generate_review_sentences`; новых контрактов нет. Смена версии модели на механизм не влияет.

**Два слоя повторения.**
- **Срочный слой** (уже был): импортированные/«интенсив»-слова (`priority`/`source==='import'`) и трудные слова (`reviewQueue`, 2+ ошибки). Всплывает немедленно, минуя даты, всегда приоритетнее спокойного. Это ответ на «слово нужно прямо сейчас» (напр. рабочая лексика пользователя).
- **Спокойный слой** (новый): обычные слова, выученные в уроках, идут по лестнице интервалов и всплывают по мере наступления даты. Держит общий словарь «живым», не мешая срочному.

**Лестница интервалов** (в календарных днях). Слово «подошло», когда `сегодня ≥ дата показа`.

| Текущая ступень | Следующий показ через |
| --- | --- |
| выучено (ступень 0) | 1 день |
| пройдена 1-дневная | 3 дня |
| пройдена 3-дневная | 7 дней |
| пройдена 7-дневная | 30 дней |
| пройдена 30-дневная | → «освоено», снят с графика |

- Верный ответ на своём показе → ступень выше, дата отодвигается.
- Ошибка на любой ступени → сброс на **день 1** и попадание в срочный слой (`reviewQueue`).
- Вершина (пройдена 30-дневная) → статус **«освоено»**, слово снимается с графика; дальше живёт в словаре и встречается только само собой (квесты, рассказы). Споткнулся об «освоенное» позже → падает на день 1 и снова в круге.

Следствие: по графику всегда крутится небольшая горстка (новые + подзабытые), а не весь накопленный словарь.

**Хранение.** Расписание держится в числовом поле `srs_due` (номер дня) — формат `today()` (ДД.ММ.ГГ) хронологически не сравнить; поле `date_review` (человекочитаемое) не переопределяется. Ступень — в `srs_stage` (0–3 на лестнице, 4 = снят). Пассивно встреченные слова (тап в сцене, глоссарий квеста) на лестницу НЕ ставятся, пока не выучены.

**Наполнение Recap** (до 7 слотов): (1) срочный слой — трудные + приоритетные/импортированные; (2) спокойный слой — подошедшие по дате, самые просроченные первыми. Переполнение: берём самые просроченные, остальные ждут следующего урока. Прежний добор «последние 10 добавленных» упразднён — день-1 расписание само подхватывает свежие слова.

**Суперквест — тоже на графике.** 80%-повторение перестаёт быть случайной выборкой: подошедшие по дате слова идут первыми, затем случайный добор. По итогу Суперквеста ступени двигаются: активно требуемые (`must_use`) слова при 0 ошибок → выше, ошибка (даже вне `must_use`, но среди подошедших по дате) → сброс. *Нюанс:* по дате поданное, но лишь пассивно мелькнувшее в сцене слово по ступени не двигается вверх (двигаются только `must_use`) — чтобы не засчитывать непроверенное; при перекосе на тесте — уточнить.

**Миграция.** Существующий словарь массово на лестницу НЕ переносится: слова, выученные до v65, попадают на неё по мере того, как их снова касается повторение; исторически трудные — через срочный слой. Backfill старого словаря — на будущее, по итогам тестирования.

**Пороговые интервалы** 1/3/7/30 — одна таблица в коде (`SRS_GAP`); методика самостоятельно обкатывается на тестировании, пороги легко переиграть.

## 2.8. Профили (несколько профилей на устройстве)

Один профиль = одна языковая пара. Несколько профилей — у каждого свои Профиль, Словарь, Свод правил, Прогресс, Аналитика. При запуске с одним профилем — вход сразу; с несколькими — экран выбора. Экспорт — вся база одним файлом.

## 2.9. Импорт своего словаря и режим интенсива

Пользователь вставляет список слов (copy-paste). Приложение разбирает построчно, дозапрашивает у Gemini формы и эмодзи, помечает «приоритетные». Режим интенсива включается автоматически: приоритетные термины отрабатываются в заданиях и Суперквестах; по мере усвоения (80–90%) подмешиваются обычные темы.

## 4.2. Шаг 2. Placement Test v2 (версия 4.17)

### Назначение и ограничения предыдущей версии

Предыдущая версия теста (v1, реализована в v31) запрашивала 2 вопроса на уровень (per_level: 2) — итого максимум 10 вопросов на уровни A1–C1. «Не знаю» и явно неверный выбор считались одинаково, что приводило к остановке теста после первой же пары «не знаю + ошибка». Лексические пробелы (незнание конкретного слова) не отделялись от грамматических ошибок. Результат был нестабилен.

Версия 4.17 исправляет эти недостатки тремя независимыми правками.

### Изменение 1. Объём — 3 вопроса на уровень

per_level изменено с 2 на 3. Итого до 15 вопросов на уровни A1–C1. Тест по-прежнему может завершиться досрочно при плохих результатах.

### Изменение 2. Разделение типов вопроса: grammar / vocab

Каждый вопрос получает поле skill: "grammar" | "vocab".

**grammar-вопросы** (2 из 3 на уровень) — проверяют структуру языка: порядок слов, спряжение, артикли, падежи, времена. ОБЯЗАТЕЛЬНОЕ ПРАВИЛО: лексика в таком вопросе берётся из уровня НИЖЕ тестируемого (т.е. вопрос по грамматике A2 использует слова уровня A1). Это гарантирует, что незнакомое слово не маскируется под незнание правила.

**vocab-вопросы** (1 из 3 на уровень) — проверяют узнавание лексики текущего уровня. Участвуют в определении итогового уровня, но НЕ участвуют в счётчике досрочной остановки — незнание слова не останавливает тест.

### Изменение 3. Раздельные счётчики: wrongChoice и dontKnow

Вместо одного счётчика wrongAtLevel[level] вводятся два:

- wrongChoice[level] — пользователь осознанно выбрал неверный вариант ответа
- dontKnow[level] — пользователь нажал «Не знаю»

**Правило досрочной остановки (строже прежнего только для grammar-вопросов):**
Тест останавливается на уровне L, если выполняется хотя бы одно из условий:
- wrongChoice[L] >= 2 (два явных неверных выбора — сильный сигнал незнания)
- wrongChoice[L] + dontKnow[L] >= 3 (сумма ошибок и «не знаю» ≥ 3 из 3 вопросов уровня)

Одно «Не знаю» в сочетании с одним верным ответом — тест продолжается.

### Порог подтверждения уровня — большинство (2 из 3)

Уровень L считается подтверждённым, если passed[L] >= 2 (то есть не менее 2 верных ответов из 3). Прежнее правило «хотя бы 1 из 2» было слишком мягким.

### Маршрутизация по результату

- Passed A1 (2+ верных на A1, тест не остановлен на A1): → A1.1.01 или выше
- Passed['A1'] === 0 (ни одного верного на A1): → A0.1.01 (уровень Pre-A1)
- Ошибка API / нет сети: → A1.1.01 (как было, fallback)

### Обновлённый контракт Gemini для placement_test

Запрос: { "task": "placement_test", "target_lang": "de", "native_lang": "ru", "levels": ["A1","A2","B1","B2","C1"], "per_level": 3 }

Ответ: { "questions": [ { "level": "A1", "skill": "grammar", "question": "Выберите правильный порядок слов: ___", "options": ["Ich arbeite heute.", "Heute arbeite ich.", "Ich heute arbeite.", "Arbeite heute ich."], "correct_index": 0 }, { "level": "A1", "skill": "vocab", "question": "Что означает слово «der Tisch»?", "options": ["стол","стул","окно","дверь"], "correct_index": 0 } ] }

Требования к grammar-вопросам: лексика строго из уровня НИЖЕ тестируемого; вопрос проверяет структурное правило, а не знание конкретного слова. Требования к vocab-вопросам: слова типичны для тестируемого уровня; все варианты в одной части речи. Ровно по 3 вопроса на уровень, по возрастанию сложности внутри уровня.

### Экран placementIntro — два пути вместо одного

Вместо одной кнопки-заглушки «Я новичок — начать с нуля (A1)» появляются ДВЕ кнопки с явным различием:

- «Не знаю язык совсем — начать с основ (A0)» → finishOnboarding(profile, 'A0')
- «Новичок, но немного знаю — пройти короткий тест» → runPlacement(profile)

Кнопка «Начать тест» остаётся как основной путь для тех, кто уверен в своём уровне.

**Обоснование двух кнопок:** человек лучше приложения знает, знает ли он вообще язык. Самодиагностика здесь надёжнее автоматического теста. Тест по-прежнему может сам направить на A0, если пользователь пошёл по пути теста, но набрал 0 на A1.

## 4.3. Уровень A0 — Pre-A1 (новый раздел, версия 4.17)

### Педагогическое обоснование

По официальной шкале CEFR (Companion Volume 2018, Совет Европы) уровень Pre-A1 зафиксирован отдельно. Он описывает человека, который: понимает отдельные знакомые слова и заученные шаблоны; может задать простейший вопрос из шаблона; не строит предложений свободно; воспринимает грамматику только как «так говорят», а не как систему. В AILEX этот уровень называется A0 и является подготовительным этапом перед A1 — без экзамена и без полноценного цикла 21-задания.

### Кто попадает на A0

- Пользователь нажал кнопку «Не знаю язык совсем» в онбординге
- Placement test вернул passed['A1'] === 0 (ни одного верного на A1)

### Структура контента A0 — 9 тематических блоков

Принцип situated learning («обучение в контексте»): с блока 3 каждый урок завершается мини-сценой — коротким диалогом (3-5 реплик) в реальной бытовой ситуации. Нарратив на родном языке; ключевые фразы встроены на целевом. Задача пользователя — не создавать текст, а выбрать готовую фразу из предложенных вариантов или тапнуть для озвучки. Фонетические блоки (1-2) мини-сцены не получают — на этом этапе человек ещё не воспринимает связный текст.

**Блок 1 — Звуки и чтение: алфавит (1-2 урока)**
Полный алфавит целевого языка с транскрипцией. Отображается тапабельной сеткой (все буквы сразу); тап → TTS. Упражнения: ТОЛЬКО фонетические — буква→звук (6 шт.) и звук→буква (6 шт.), итого 12. Лексические вопросы («что значит слово?») запрещены — человек ещё не знает языка. Режим Gemini: mode="alphabet".

**Блок 2 — Звуки и чтение: буквосочетания (1-2 урока)**
Диграфы и трудные сочетания (sch, ch, ei, ie, ü, ö, ä для немецкого и т.п.). Карточки с примерами и 12 упражнениями. Режим Gemini: mode="alphabet".

**Блок 3 — Первый контакт (1 урок) 🎭 мини-сцена**
Приветствия, прощания, «как вас зовут», «меня зовут», «откуда вы», «я из...». Chunk learning — заученные формулы целиком. Мини-сцена: знакомство с незнакомым человеком.

**Блок 4 — Числа и цвета (1 урок)**
Числа 1-100, основные цвета. Без мини-сцены — лексика тренируется упражнениями на узнавание.

**Блок 5 — Время и даты (1 урок) 🎭 мини-сцена**
Сегодня/завтра/вчера, утро/день/вечер/ночь, дни недели, месяцы. Мини-сцена: спросить время или договориться о встрече.

**Блок 6 — Мой мир (1 урок)**
Члены семьи, 5-6 базовых профессий, предметы вокруг (телефон, ключ, деньги, сумка, дом, машина). Без мини-сцены — лексический урок.

**Блок 7 — Еда, тело и самочувствие (1-2 урока) 🎭 мини-сцена**
Еда и напитки (выживательный минимум: вода, хлеб, кофе, суп, яблоко), части тела (голова, рука, нога, живот, спина), самочувствие (болит, плохо, хорошо). Мини-сцена: аптека или приём у врача.

**Блок 8 — Ориентация и магазин (1-2 урока) 🎭 мини-сцена**
Вопросительные слова (где? что? как? когда? сколько?), направления (направо/налево/прямо), транспорт (базовый), магазин и цены («Сколько стоит?», «Дайте, пожалуйста», «Спасибо»). Мини-сцена: спросить дорогу или купить что-то.

**Блок 9 — Первые предложения (1-2 урока — финал A0) 🎭 мини-сцена**
Самые базовые конструкции: «Я + существительное/прилагательное» (Ich bin Mechaniker), «Это + существительное» (Das ist ein Tisch), «У меня есть» (Ich habe), «Я хочу» (Ich möchte). Мини-сцена: знакомство + короткий рассказ о себе. Мост к A1.

### Коды уроков и переход

- Коды: A0.1.01, A0.1.02, … A0.1.NN (подуровень всегда 1)
- Схема «2 подуровня × 25 уроков» применяется только с A1 — A0 вне неё
- После последнего урока A0: автоматический переход на A1.1.01
- Экзамена нет: A0 — подготовительный этап, не CEFR-уровень
- Запись в прогресс: обычная строка A0.1.NN → A0.1.(NN+1) или A1.1.01 для последнего

### Принципы методики A0

- Транскрипция — центральный элемент каждого шага, а не вспомогательный
- TTS — обязательно для каждого звука/слова/шаблона; слушать первично
- Никакого production (свободного написания) до блока 9
- Все задания — узнавание: тап, выбор варианта, сопоставление
- Объяснения строго на родном языке (native_lang), как и везде
- Situated learning (с блока 3): мини-сцена в конце урока показывает изученные фразы в реальной бытовой ситуации; задача — выбор готовой фразы или её озвучка, не свободное построение
- Gemini генерирует контент через контракт a0_lesson (Пример F, раздел 2.5); решение о переходах — за приложением

### Что A0 НЕ включает

- Экзамен (нет)
- Возврат из главного меню (A0 — одноразовый онбординговый путь)
- Суперквест на уровне A0 (нет — слишком мало материала)
- Полный цикл 21-задания (формат другой — короче и легче)

### Генерация Ядра языка (раздел 4.26)

Сразу после анкеты интересы пользователя уже известны — поэтому Ядро генерируется здесь, а не заранее и не на сервере. Порядок:

1. **Закрытые классы целиком** — местоимения, предлоги, союзы, частицы, артикли, неправильные глаголы. Для немецкого ≈300 слов ≈ 3 запроса.
2. **Открытый класс уровня онбординга** — ≈600 слов для A1 ≈ 5 запросов, в пропорции уровня (A1 — 40% бытовое / 60% интересы, доля берётся из раздела 5).
3. Итого ≈8 последовательных запросов, ориентировочно 1–3 минуты с экраном ожидания и честной подписью «собираю словарь языка — это один раз».

**Догрузка следующего уровня** идёт фоном, пока человек учит текущий: триггер — либо пройдено ~70% уроков уровня, либо в Ядре осталось меньше 100 невыданных слов открытого класса. Пользователь ожидания не видит.

**Миграция существующих профилей.** Профиль, созданный до 4.26, Ядра не имеет. При первом запуске билда v64 приложение генерирует Ядро разово, с тем же экраном ожидания; уже накопленный Словарь при этом целиком уходит в список `exclude` — выученное повторно в Ядро не попадает.

**Смена интересов в профиле.** Перегенерируется только тематический слой (100–200 слов, 1–2 запроса, секунды). Общий слой и закрытые классы не трогаются. Невыданные слова прежней темы → «архив» (раздел 2.3-Я).

**Режим интенсива Ядра не касается.** Импортированный пользовательский словарь (раздел 2.9) работает как работал: приоритетные термины идут в порцию первыми, помимо Ядра.

# 5. Методология урока (The 21-Drill System)

Применяется для уровней A1 и выше. Уровень A0 использует собственный более простой формат (раздел 4.3); после перехода на A1 — полноценный 21-drill.

## 5.0. Глобальные директивы

- **Соответствие CEFR:** весь контент строго соответствует текущему уровню урока.
- **Родной контекст:** вся теория, объяснения и подсказки — строго на родном языке, с мнемотехниками.
- **No back-translation:** в упражнениях запрещено просить перевод с целевого языка на родной.

### Чтение и звучание (транскрипция + озвучка)

- Обязательная транскрипция родными буквами для каждого нового слова (например [хэлоу]).
- Трудные слова повторяются с транскрипцией при каждом повторе.
- Озвучка (TTS) доступна для слов и фраз — кнопка прослушивания.
- Честная оговорка: транскрипция приблизительна. Это опора для чтения, а не эталон.

### Движок квестов и рассказов (Quest Core)

Атмосфера строится на интересах пользователя из профиля (не фиксированная НФ-стилистика). Anti-Frustration: после 2 ошибок AILEX даёт правильный ответ через мнемотехнику и завершает квест.

### Генератор лексики (Vocabulary Engine) — редакция 4.26

**Кто выбирает слова.** Порцию урока собирает **приложение** из Ядра языка (раздел 2.3-Я) — арифметикой, без обращения к Gemini. Модель к моменту урока уже сделала свою работу (сгенерировала Ядро при создании профиля) и в уроке отвечает только за задания, правила и сцены. Это возвращает подбор лексики туда, где ему место по конституции: выбор слов — методика, методика принадлежит приложению.

**Состав порции — 7 единиц, гарантированно каждый урок:**

- **5 новых слов** открытого класса — в пропорции уровня (см. «Баланс лексики»), первыми идут слова с меньшим `freq_rank`.
- **2 неправильных глагола** из закрытого класса — не «если уместны», а всегда, пока класс не исчерпан (нем.: ~200 глаголов ≈ 100 уроков ≈ учебный год).

**Почему именно 2, а не 1** (обсуждено 17.07.26): при 200 глаголах в языке два за урок дают полный охват примерно за год при терпимой нагрузке; глагол успевает всплыть в квестах и Суперквесте прежде, чем придёт следующий. Один глагол растянул бы охват вдвое без выигрыша в запоминании.

**Гарантии, которых не было до 4.26:**

- **Порция всегда полная.** Дубликаты кончаются структурно: приложение вычёркивает выданное из Ядра, а не угадывает по последним 300 словам, отправленным модели. Недобор («пришло 3 слова вместо 7») исчезает как класс.
- **Состав предсказуем.** Часть речи, тема и уровень каждого слова известны из Ядра — их не нужно выпрашивать у модели и потом проверять.
- **Служебные слова доходят до урока.** До 4.26 союзы и предлоги были гарантированы только в Суперквесте (2 штуки на прогон); в обычном уроке их могло не быть месяцами. Теперь они выдаются из закрытого класса по мере прохождения, наравне с прочей лексикой.

**Потолок грамматики и показ форм — разные вещи (принято 17.07.26).** Потолок держится на том, что от человека **требуют в заданиях** (`known_rules`), а не на том, что ему **показывают**. Поэтому:

- **Показывать формы неправильного глагола можно всегда**, с первого дня, даже если правило (например, Perfekt) ещё не пройдено — справочно, в скобках, с пометкой «этот глагол не по правилам, запомни целиком». Минимальное пояснение вида «прошлое время — такая форма», без грамматических терминов.
- **Требовать эти формы в заданиях нельзя**, пока правило не пройдено. Задание строится строго по `known_rules`.

Обоснование то же, что у 📘 «почему так» (раздел 2.5, Пример G): справка по запросу — не давление контентом. Человек видит, что глагол особенный, задолго до того, как поймёт устройство правила; форма запоминается через контекст, а правило потом ложится на уже знакомое слово.

**Резервный путь.** Ядро исчерпано или его генерация сорвалась → порция набирается прежним способом (`generate_vocab`, раздел 2.5, Пример A) с фильтром дубликатов **и добором**: если после фильтра осталось меньше 7 слов — повторный запрос со списком исключений (известное + только что предложенное). Обучение не блокируется никогда.

### Баланс лексики: интересы + бытовое

A1 — 40% бытовое / 60% интересы; A2 — 35/65; B1 — 30/70; B2 — 25/75; C1–C2 — 20/80.

На A0 — 100% выживательная/бытовая (весь курс подготовительный).

### Структура 21 задания

- Повторение (7): повторение из предыдущего урока (термин «Recap» устарел, заменён на «Повторение» в v34)
- Presentation: таблица новых слов
- Grammar Card: объяснение нового правила (показывается только если есть новое правило)
- Задания 1–5 (Recognition): выбор варианта
- Задания 6–11 (Writing): написание по памяти
- Задания 12–20 (Construction): построение фраз и предложений (типы — см. ниже, редакция 4.28)
- Задание 21 (Final Challenge): интерактивный рассказ-квест (раздел 5.A)

**Проверка по смыслу:** равноценные формы (im = in dem, zum, am) засчитываются верными. Опечатка 1–2 символа — верно с пометкой. Ошибка только в артикле — «почти верно», не блокирует, копится в адаптивное закрепление.

**Подсказки-скелеты:** скрывают именно целевой элемент задания (артикль/спряжение/порядок/слово).

**Полная подсказка (4.14):** правильный ответ целиком с транскрипцией, переводом и озвучкой.

### Типы письменных заданий (редакция 4.28)

Три письменных типа отличаются объёмом того, что учащийся строит сам, и образуют лесенку нагрузки. Все три уже методически разведены; 4.28 добавляет «Собери фразу» как основной тип Construction на A1 и наводит порядок в условии.

- **«Слово в контексте»** (Writing, `drillContext` / `drillReviewContext`). Готовая фраза целевого языка с одним пропуском `___`; вставить одно слово. Самое лёгкое. Условие несёт перевод фразы на родной язык. Механика пропуска — редакция 4.20 (два пути проверки: одно слово либо фраза целиком с вырезанием слова по позиции).

- **«Собери фразу»** (Construction, основной тип на A1 — редакция 4.28). Даётся перемешанный набор всех слов эталонной фразы + перевод-условие на родной язык; собрать всю фразу и **напечатать самому** (не тап по плиткам — печать намеренна: она заставляет извлекать форму из памяти, сборка кликами учит медленно). Полная механика — новый подраздел 5.2.

- **Свободный ввод** (Construction, дополнительный вариант). Даётся только перевод-смысл; написать фразу с нуля. Тяжелейший из трёх; меньшая доля на A1, полезнее на уровнях выше, где есть чем варьировать.

### Тап-перевод во всех заданиях урока (редакция 4.28)

Жест «тапни незнакомое слово целевого языка → перевод и транскрипция; для незнакомого доступно 📘 „почему так“» распространяется с одних только сцен Квеста/Суперквеста (где он жил с v41–v57) на **весь урок**: «Слово в контексте», «Собери фразу», свободный ввод, повторение в контексте, «Фразы на каждый день», а также на **экран разбора ошибки** (правильный ответ и примеры в правиле).

Границы жеста неизменны: тап переводит одно отдельное слово по запросу и **ничего не выдаёт раньше времени** — готовый ответ по-прежнему раскрывают лампу/разбор, а прогрессивное раскрытие («сначала попробуй сам») остаётся. Тап — добровольная опора, а не подсказка-ответ. Технически переиспользуются готовые `tapAllWordsHTML` + `attachTapAll` + `lookupAndShowWord` (порядок поиска: слова задания → словарь → Gemini; «почему так» доступно, когда слово внутри контейнера `.target`). Служебные слова кликабельны наравне с прочими (набор «Собери фразу» намеренно включает их — см. 5.2).

## 5.A. Интерактивный рассказ-квест с развилками

Гибридный движок: жёсткий каркас (приложение) + живая реакция (Gemini). Структура: 2 развилки по 2 варианта каждая. Масштабирование по CEFR: A1 — 4–6 предложений; A2 — 7–10; B1 — 12–15; B2 — 16–20; C1–C2 — 20+.

## 5.B. Голосовой диалог (опция «Поговорить»)

Цепочка STT → Gemini → TTS. Двойной ввод (голос/текст). Роль «напарник, а не судья». Метрика фиксируется в аналитике.

## 5.1. Протокол сохранения

Автозапись после последнего задания: строка в «Прогресс», новые слова в «Словарь», метрики в «Аналитику». Уведомление «Сохранено ✓».

## 5.2. Задание «Собери фразу» (word-bank) — редакция 4.28

**Назначение.** Учит порядку слов и активному построению фразы из готовых кирпичей, снижая барьер на слабом уровне. Основной тип Construction на A1; свободный ввод (только перевод-смысл → написать с нуля) остаётся дополнительным вариантом, полезнее на уровнях выше.

**Повод.** Тест A1.1.11 на v65 вскрыл в Construction три болячки разом (скриншоты 21.07.26): (1) смешанный язык в условии — русский текст с непереведёнными целевыми словами («Мне нужен новый рюкзак для Weltreise»), в пределе весь целевой эталон целиком в поле условия («Die Kontrolle läuft bei der Produktion.» как «задание»); (2) готовый ответ выдан прямо в условии — строить нечего; (3) неизученные слова в задании (Werkstor, Weltreise), которых нет в словаре. Корень (1) и (2) — в том, что условие рисуется из `prompt_native`, а Gemini кладёт туда целевые слова/эталон вопреки промпт-правилу симметрии (та же природа, что QUEUE-09/34).

**Механика.** Условие — короткая инструкция + перевод целевой фразы на родной язык (что сказать), строго без единого слова целевого языка и без самой целевой фразы. Под условием — набор всех слов эталона, включая служебные (артикли, предлоги, союзы), перемешанный, через запятую. Учащийся печатает фразу сам — не тап по плиткам.

**Граница логика/контент (ключевое решение).** Набор слов формирует **приложение** из `expected_target` — разбивает на слова и перемешивает. Это арифметика приложения, не Gemini. Отсюда две гарантии кодом, а не уговорами модели: набор всегда содержит все слова, включая служебные (иначе фразу физически не собрать), и целевой фразы в условии не появляется в принципе — она живёт только в наборе и эталоне. Промпт-правило симметрии остаётся, но перестаёт быть единственной защитой от смешанного языка: даже если Gemini снова его нарушит, условие показывает только `prompt_native`, а фраза не выводится.

**Тап-перевод.** Каждое слово набора кликабельно — перевод + транскрипция, для незнакомого доступно 📘 «почему так» (общий механизм 4.28, см. «Тап-перевод во всех заданиях урока»). Тап на всех словах, включая служебные.

**Лексика.** Эталон строится из пройденных слов; **не более 1 нового слова на фразу**, и это новое обязано быть в наборе с переводом (добирается тапом, а не блокирует). На типичной A1-фразе (4–6 слов) это ~20% — тот «какой-то процент», при котором фразу всё ещё можно собрать. Основной поток новой лексики идёт через порцию урока (5+2 из Ядра, раздел 5.0); «Собери фразу» её закрепляет, а не вываливает новую пачку.

**Проверка** — прежняя, мягкая по смыслу: синонимы, опечатки 1–2 символа, ошибка только в артикле не блокируют (общая «проверка по смыслу», раздел 5).

**Правки контракта `construct`** (внутренний контракт заданий 12–20, в спеке отдельным Примером не выделен): (а) `prompt_native` — 100% на родном языке, явный запрет любого слова целевого языка; (б) `expected_target` — из известных слов, ≤1 нового; новое обязано попасть в `glossary`; (в) `glossary` — все слова эталона, включая служебные, каждое с переводом и транскрипцией, чтобы тап работал на любом слове набора.

**Отклонено:** тап по плиткам-словам вместо печати (упрощает до сборки кликами — «простые задачи ничему не учат или учат очень медленно»); чистка условия построчной код-сверкой слов (хрупко, как отклонённый ранее пост-фильтр QUEUE-09) — решено архитектурно тем, что набор строит приложение, а фраза в условие не выводится.

# 6. Суперквест

**Статус:** сюжетный движок реализован в коде (билд v55). Единый квест-погружение на 30–50 сцен, разбитый на акты — связная история-новелла, в которую вплетена лексика.

**Принцип (главный):** Суперквест — новелла, которую интересно читать: сквозной сюжет, живой герой от первого лица, смена мест и действий, юмор и яркие образы, развилки, после которых хочется узнать, что будет с героем. Иерархия приоритетов: (1) динамика и связность сюжета, (2) иммерсия в образ героя, (3) повторение лексики — вторично и подчинено сюжету. CEFR ограничивает ТОЛЬКО грамматику задания (на A1 — простое предложение), но не богатство мира, образов и словаря сцены.

## 6.1. Двухслойная генерация

Корень прежней репетативности: набор `must_use` считался один раз на акт и передавался в каждую сцену с требованием использовать преимущественно эти слова — при статичном сценарии Gemini пересказывал ту же ситуацию. Решение — разнести «что происходит» и «как написано».

**Слой 1 — Арка** (task `generate_story_arc`, один вызов на старте суперквеста): возвращает скелет истории — `logline`, уникальный `hero {name, role, voice}`, `tone` и список `beats` по числу сцен. Бит = `{place, action, shift, опц. twist}`; в арке заранее размечены точки развилок (1–2 на акт). Хранится в `L.superStory.arc`, переживает все акты.

**Слой 2 — Сцена** (task `quest_step`, по одному биту за раз): получает текущий `beat` + `hero` + `tone` + `story_so_far` + выбранную ветку и разворачивает ИМЕННО этот бит в живую сцену, не описывая ситуацию заново. Смена сцен гарантируется структурой арки, а не уговорами в промпте.

## 6.2. Лексика — мягкий приоритет

`must_use` и `vocab_balance` — мягкая опора, не закон: словарные слова вплетаются там, где естественно по сюжету; новые слова допустимы и желательны для живости истории. Закрепление достигается тем, что приоритетные слова распределены аркой по битам (у каждого естественный повод появиться); тап-перевод и глоссарий подхватывают любое новое слово в живом контексте. **Отбор слов повторения (4.27):** сначала подошедшие по интервальному графику (раздел 2.7-И), затем срочный слой (импорт/интенсив + трудные), затем случайный добор — вместо прежней чисто случайной выборки; по итогу Суперквеста ступени слов двигаются (см. 2.7-И).

## 6.3. Структура: акты и биты

| Уровень | Число актов | Общий объём |
| --- | --- | --- |
| A1 | 3 акта | ~30 сцен |
| A2 | 3 акта | ~34 сцены |
| B1 | 4 акта | ~40 сцен |
| B2 | 4 акта | ~44 сцены |
| C1–C2 | 5 актов | ~50 сцен |

Число битов = число сцен; биты раздаются по актам тем же механизмом, что и сцены (`splitScenesAmongActs`). Между актами — экран-итог («Глава N пройдена»).

## 6.4. Развилки

Развилка влияет на путь героя (арка планирует последствие выбора), варианты переводятся (`label_native`) и создают вопрос «что будет дальше». Граница — 1–2 на акт. Выбор переносится в `story_so_far` (choiceNote) и учитывается в последующих битах.

**Развилка — сюжетная, не грамматическая (QUEUE-48, спека 4.27).** Два варианта `choices` — это РАЗНЫЕ поступки/повороты сюжета (что герой делает дальше), ведущие к разным продолжениям. Оба варианта грамматически верны и естественны. Запрещено превращать развилку в грамма-квиз (выбор между верной и неверной формой — артикль/падеж/окончание/спеллинг — или между двумя формулировками одного и того же): `label_target` — осмысленное действие в мире истории, а не языковой тест. Языковое задание сцены (`expected_target`) отдельно от выбора ветки. Правило закреплено в промптах `quest_step` (Квест и Суперквест) и в разметке развилок `generate_story_arc` (`branch_hint`).

## 6.5. Иммерсия, тон, задание

Повествование от первого лица, постоянный голос героя через все сцены и акты. Тон — живой, с юмором и яркими сенсорными образами, в рамках CEFR-грамматики (образность простыми словами). Пользователь отвечает как герой. `prompt_for_user` НЕ выдаёт готовый эталон целиком — структура-подсказка предъявляется только как `scaffold` с прочерками.

## 6.6. Тап-перевод новых слов

Слова из `encountered_words` кликабельны в тексте. Тап → карточка (перевод + транскрипция + озвучка). Поиск приоритезирует слова текущей сцены. Дополнение к списку 📒 «Слова задания», не замена.

## 6.7. Anti-Frustration на длинной дистанции

Счётчик напарника обнуляется в начале каждого акта. Счётчик итоговой точности копится за весь Суперквест без сброса.

## 6.8. Производительность и подгрузка

Арка строится один раз на старте под лоадером «Готовлю историю…». Следующий акт готовится в фоне, пока пользователь проходит текущий.

## 6.9. Завершение и сохранение

Запись: `SUPER_[код]`. Итоговая точность — по накопленным за весь Суперквест данным.

## 6.10. Контракт Gemini

`generate_story_arc` → `{logline, hero{name,role,voice}, tone, beats:[{act,scene,place,action,shift,twist,is_branch_point,branch_hint}]}` (число beats = число сцен).
`quest_step` (сцена) дополнен входными полями `beat`, `hero`, `story_tone`; `must_use`/`vocab_balance` переформулированы как мягкие. Graceful fallback: при сбое генерации арки суперквест работает в прежнем режиме «сцена за сценой» по `story_so_far`.

- Первый провал: EXAM_[Уровень]_FAIL-1, возврат к экзамену.
- Второй провал: переход в режим работы над ошибками (раздел 8).

# 8. Режим работы над ошибками (Remedial)

Активируется после второго провала экзамена.

- Разбор: объяснение ошибок на родном языке с мнемотехниками.
- Тренинг: по 2 штрафных задания на каждую ошибку.
- После тренинга: запись EXAM_[Уровень]_REMEDIAL → возврат к экзамену.

# 9. Визуализация и мнемоника

## 9.1. Графический стандарт

Маркеры: 💡 логика, 🎯 задание, 🌍 культура, 🧠 запомни, ✅ / ❌, 💾 сохранение. Транскрипция обязательна для каждого нового слова. TTS доступна для слов и фраз при наличии поддержки браузером; при отсутствии озвучки — только транскрипция. Не более одного задания на экран.

## 9.2. Универсальные визуальные мнемоники

Формат: [слово] → [эмодзи] → [форма / перевод]. Примеры: trinken → 🥤 → getrunken = пить; sehen → 👀 → gesehen = видеть.

- «Повторение» — блок повторения лексики из прошлых уроков.

- «встречено в контексте» — слово встретилось в сцене/задании, но не изучалось активно.
- «к изучению» — слово в очереди на изучение.
- «трудное» — слово, в котором были ошибки (в очереди повторения).

## Подсказки экранов и справка (переводятся как целые абзацы)

- Подсказка «Новые слова» — как пользоваться экраном новых слов.
- Подсказка «Задания» — как отвечать на упражнения.
- Подсказка «Квест» / «Суперквест» — как проходить историю, тап по слову = перевод.
- Подсказка «Словарь» — как открыть карточку слова.
- Подсказка «Аналитика» — как читать прогресс.
- Справка «Как пользоваться» — единый свод жестов (тап по слову, звук, лампочка, правило, карточка словаря, микрофон).

## Потребители локализации (новые элементы 4.25)

- **«Показать подсказки заново»** (Настройки) — сбрасывает отметки `hint_shown_*`, чтобы одноразовые подсказки экранов появились снова. Провод к уже существующей, но не подключённой функции `Hints.reset()`.
- **Экран «Как пользоваться»** (Настройки) — компактный свод жестов: тап по слову = перевод и «почему так»; кнопка звука = повтор озвучки; лампочка = подсказка; линейка = правило; тап в словаре = карточка слова; микрофон в диалоге = голосовой ответ. Текст переводится через путь «целые абзацы».
