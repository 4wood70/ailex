# 1. Реализовано в основной версии (index.html v41, работает)

- Цикл урока из 21 задания (термин «дрилл» устаревший, заменяется на «задание»).

- Placement Test v2 (реализован в v33): 3 вопроса на уровень × 5 уровней = до 15 вопросов; поле skill: grammar | vocab; раздельные счётчики wrongChoice / dontKnow; остановка при wrongChoice>=2 ИЛИ wrongChoice+dontKnow>=3; порог подтверждения 2 из 3; маршрут passed['A1']=0 → A0.1.01. (v1 с 2 вопросами — устарел, заменён.)

- Нарративные квесты с развилками (2 ветки × 2 выбора).

- Уровень A0 — Pre-A1 (реализован в v33): 9 тематических блоков (~12–15 уроков по дистанции письменности); контракт Gemini a0_lesson с items/mini_check/mini_scene; алфавитные блоки (1–2) — только фонетические упражнения (буква↔звук, 12 шт.), без лексики; блоки 3–9 — situated learning с мини-сценами; автопереход на A1.1.01 после последнего урока; Суперквест на A0 недоступен.

- Суперквест (заменяет «Супер-урок», v31) — единый квест-погружение на 30–50 предложений: 3–5 актов по CEFR с экраном-итогом между ними, гибкие развилки (не больше 1–2 на акт), тап-перевод новых слов, Anti-Frustration по актам, итоговая точность за весь Суперквест. Проверено сквозным прогоном.

- Словарь с мнемониками тир 1 (текст + эмодзи).

- Экран аналитики / прогресса (кольцо по урокам уровня, точность отдельной подписью).

- Голосовой диалог (STT → Gemini → TTS).

- Движок адаптации «Gemini советует — приложение решает».

- Локальная база на устройстве (AILEX_StatusDB); экспорт/импорт через JSON.

- Несколько профилей на устройстве (один профиль = одна языковая пара).

- Перебор моделей Gemini с устойчивостью к сбоям и лимитам 429 (thinkingBudget = 0, запоминание рабочей модели).

- PWA-оболочка (manifest, sw.js с версионированием кэша); APK собран через PWABuilder (TWA); опубликовано на GitHub Pages.

- Расширенный список языков (v35): 72 языка (было 15) — все языковые пары, которые Gemini знает достаточно для обучения. Поле bcp у каждого языка: BCP-47 для TTS, либо null если надёжной озвучки нет.

- TTS с деградацией без ошибок (v35): если у целевого языка bcp=null — кнопки 🔊 скрываются везде (словарь, Grammar Card, квест, суперквест, sayBtn); speak() и makeRecognizer() молча возвращают управление; транскрипция родными буквами остаётся единственной опорой. Текст подсказки на экране новых слов адаптируется («транскрипция — ваша главная опора» вместо «эталон даёт озвучка»).

- Виджет выбора языка с поиском (v35): оба экрана выбора языка (родной и изучаемый) теперь отображают поле фильтрации — фильтрует список по мере набора (по названию или коду языка). select заменён на кастомный список lang-opt с hidden input. Дефолтные значения подставляются автоматически.

- Механика заданий с пропуском v1 (v36, спецификация 4.20): drillReviewContext и drillContext принимают полное предложение; формулировка «Напишите предложение, подставив пропущенное слово»; два пути проверки: (1) полное предложение → вырезать слово по позиции пропуска, (2) одно слово → засчитать + показать полное предложение; после верного ответа — полное предложение + 🔊.

- Фикс multi-word extraction (v37): баг извлечения заполненного слова из полного предложения для многословных ответов (например, «das Hotel») — исправлен.

- Фикс answer key completeness (v38): промпт Gemini получил явное требование — answer_key должен быть грамматически полным предложением на целевом языке (с артиклями, падежами и т.д.). Ранее Gemini мог опускать артикли, что вызывало ложные ошибки у пользователя.

- Фикс дистракторов в drillRecog (v38): фильтр на seen=true, длина < 40 символов, отсутствие кириллицы и скобок/стрелок в поле term — исключает несмотренные импортированные слова и заголовки словарей («СЛОВАРЬ ТЕРМИНОВ ЧПУ OKUMA (нем. → рус.)»).

- Инструкция в drillRecog (v38): добавлена строка «🎯 Какое слово соответствует: «…»?» — задание стало явным.

- Попап тап-слова (v39): добавлена кнопка ✕ для явного закрытия (ранее закрывался только тапом вне попапа). Закрытие тапом по фону сохранено как дополнительное.

- Транскрипция родными буквами в квестах (v39): промпты quest_step для квеста и суперквеста ужесточены — явный запрет IPA, обязательная транскрипция символами native_lang с конкретным примером [форбэрайтн]. Gemini больше не может подставлять IPA-символы.

- Расширенный пул запасных моделей Gemini (v40): было 2 модели, стало 6 запасных — приложение перебирает их по очереди с паузой между попытками, без агрессивного спама к API.

- Фикс Суперквеста при обрыве соединения (v40): Суперквест больше не засчитывается как «пройденный» если соединение оборвалось до начала первой сцены. Показывается экран «Нет связи — попробовать снова».

- Компактный попап тап-слова (v41): убрано затемнение всего экрана при тапе на слово. Попап теперь — тонкая плашка внизу экрана, фон остаётся виден. Формат: «слово 🔊 · [транскрипция] · перевод · ✕». TTS-кнопка сохранена в компактном виде (иконка рядом со словом, не кнопка на всю ширину).

- Тап-перевод всех слов в квестах и суперквестах (v41): ранее подсвечивались только новые слова из newGloss. Теперь все значимые слова кликабельны (кроме служебных: артикли der/die/das/ein, союзы und/oder/aber и т.п.). Логика: сначала поиск в словаре пользователя (мгновенно) → если нет, запрос к Gemini (~1 сек с индикатором) → новое слово автоматически сохраняется в словарь со статусом «встреченное». Встреченные слова текущего урока кэшируются в L.encounteredCache для быстрого повторного поиска.

- Онбординг-подсказки (v41): система одноразовых оверлеев при первом входе на экран. Флаги хранятся в localStorage (ключи hint_shown_*). Закрываются кнопкой «Понятно ✓» или тапом по фону. Покрыто 6 экранов: новые слова урока, задания (drill), квест, суперквест, словарь, главный экран (прогресс-кольцо). Тексты подсказок в объекте HintTexts.

# 3. Зафиксировано в спецификации, внедряется в код поэтапно (4.9–4.14)

- Паспорт слова: формы глаголов (инфинитив/претеритум/партицип II) и существительных (артикль рода, мн. число).

- Точное правило под конкретную ошибку, показывается сразу.

- Прогресс по уровню CEFR; переход на уровень — через экзамен.

- Каталог словаря по первой букве слова без артикля; фильтр «только неправильные глаголы».

- Человечная проверка по смыслу (синонимы, опечатки, артикли не блокируют; подсветка места ошибки).

- Подсказки-скелеты с угасанием, прячущие целевой элемент задания.

- Полная подсказка: правильный ответ с транскрипцией, переводом и озвучкой.

- Слой повседневных выражений (Alltagsphrasen) — фразы-клише с переводом, транскрипцией, контекстом, озвучкой.

- Импорт своего словаря (copy-paste) и автоматический режим интенсива под профессиональную лексику.

- Локализация интерфейса по системному языку с автопереводом через Gemini и кэшем.

- Идиоматичная локализация терминов (реализована в v34): промпт передаёт Gemini контекст и функцию каждого термина (таблица UI_CTX), Gemini подбирает идиоматичное слово языка — не дословный перевод. «Recap» переименован в «Повторение» во всём коде.

# 8. Базовые принципы (рамка, неизменно)

- Логика → приложение, контент → Gemini, хранилище → локальная база на устройстве.
- Смена версии модели Gemini НЕ меняет методику обучения.
- Gemini — только генератор контента: слова, фразы, проверка, квесты — всегда строго в JSON по контракту. Методику не знает и не определяет.
- Принцип терминологии: методические термины в интерфейсе — на родном языке учащегося.
