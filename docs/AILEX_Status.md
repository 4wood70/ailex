# 1. Реализовано в основной версии (index.html v67)

- Цикл урока из 21 задания (термин «дрилл» устаревший, заменяется на «задание»).

- Placement Test v2 (реализован в v33): 3 вопроса на уровень × 5 уровней = до 15 вопросов; поле skill: grammar | vocab; раздельные счётчики wrongChoice / dontKnow; остановка при wrongChoice>=2 ИЛИ wrongChoice+dontKnow>=3; порог подтверждения 2 из 3; маршрут passed['A1']=0 → A0.1.01. (v1 с 2 вопросами — устарел, заменён.)

- Нарративные квесты с развилками (2 ветки × 2 выбора).

- Уровень A0 — Pre-A1 (реализован в v33): 9 тематических блоков (~12–15 уроков по дистанции письменности); контракт Gemini a0_lesson с items/mini_check/mini_scene; алфавитные блоки (1–2) — только фонетические упражнения (буква↔звук, 12 шт.), без лексики; блоки 3–9 — situated learning с мини-сценами; автопереход на A1.1.01 после последнего урока; Суперквест на A0 недоступен.

- Суперквест (заменяет «Супер-урок», v31) — единый квест-погружение на 30–50 предложений: 3–5 актов по CEFR с экраном-итогом между ними, гибкие развилки (не больше 1–2 на акт), тап-перевод новых слов, Anti-Frustration по актам, итоговая точность за весь Суперквест. Проверено сквозным прогоном.

- Словарь с мнемониками тир 1 (текст + эмодзи).

- Экран аналитики / прогресса (кольцо по урокам уровня, точность отдельной подписью).

- Голосовой диалог (STT → Gemini → TTS).

- Движок адаптации «Gemini советует — приложение решает».

- Локальная база на устройстве (AILEX_StatusDB); экспорт/импорт через JSON.

- Несколько профилей на устройстве (один профиль = одна языковая пара).

- Перебор моделей Gemini с устойчивостью к сбоям и лимитам 429 (thinkingBudget = 0, запоминание рабочей модели).

- PWA-оболочка (manifest, sw.js с версионированием кэша); APK собран через PWABuilder (TWA); опубликовано на GitHub Pages.

- Расширенный список языков (v35): 72 языка (было 15) — все языковые пары, которые Gemini знает достаточно для обучения. Поле bcp у каждого языка: BCP-47 для TTS, либо null если надёжной озвучки нет.

- TTS с деградацией без ошибок (v35): если у целевого языка bcp=null — кнопки 🔊 скрываются везде (словарь, Grammar Card, квест, суперквест, sayBtn); speak() и makeRecognizer() молча возвращают управление; транскрипция родными буквами остаётся единственной опорой. Текст подсказки на экране новых слов адаптируется («транскрипция — ваша главная опора» вместо «эталон даёт озвучка»).

- Виджет выбора языка с поиском (v35): оба экрана выбора языка (родной и изучаемый) теперь отображают поле фильтрации — фильтрует список по мере набора (по названию или коду языка). select заменён на кастомный список lang-opt с hidden input. Дефолтные значения подставляются автоматически.

- Механика заданий с пропуском v1 (v36, спецификация 4.20): drillReviewContext и drillContext принимают полное предложение; формулировка «Напишите предложение, подставив пропущенное слово»; два пути проверки: (1) полное предложение → вырезать слово по позиции пропуска, (2) одно слово → засчитать + показать полное предложение; после верного ответа — полное предложение + 🔊.

- Фикс multi-word extraction (v37): баг извлечения заполненного слова из полного предложения для многословных ответов (например, «das Hotel») — исправлен.

- Фикс answer key completeness (v38): промпт Gemini получил явное требование — answer_key должен быть грамматически полным предложением на целевом языке (с артиклями, падежами и т.д.). Ранее Gemini мог опускать артикли, что вызывало ложные ошибки у пользователя.

- Фикс дистракторов в drillRecog (v38): фильтр на seen=true, длина < 40 символов, отсутствие кириллицы и скобок/стрелок в поле term — исключает несмотренные импортированные слова и заголовки словарей («СЛОВАРЬ ТЕРМИНОВ ЧПУ OKUMA (нем. → рус.)»).

- Инструкция в drillRecog (v38): добавлена строка «🎯 Какое слово соответствует: «…»?» — задание стало явным.

- Попап тап-слова (v39): добавлена кнопка ✕ для явного закрытия (ранее закрывался только тапом вне попапа). Закрытие тапом по фону сохранено как дополнительное.

- Транскрипция родными буквами в квестах (v39): промпты quest_step для квеста и суперквеста ужесточены — явный запрет IPA, обязательная транскрипция символами native_lang с конкретным примером [форбэрайтн]. Gemini больше не может подставлять IPA-символы.

- Расширенный пул запасных моделей Gemini (v40): было 2 модели, стало 6 запасных — приложение перебирает их по очереди с паузой между попытками, без агрессивного спама к API.

- Фикс Суперквеста при обрыве соединения (v40): Суперквест больше не засчитывается как «пройденный» если соединение оборвалось до начала первой сцены. Показывается экран «Нет связи — попробовать снова».

- Компактный попап тап-слова (v41): убрано затемнение всего экрана при тапе на слово. Попап теперь — тонкая плашка внизу экрана, фон остаётся виден. Формат: «слово 🔊 · [транскрипция] · перевод · ✕». TTS-кнопка сохранена в компактном виде (иконка рядом со словом, не кнопка на всю ширину).

- Тап-перевод всех слов в квестах и суперквестах (v41): ранее подсвечивались только новые слова из newGloss. Теперь все значимые слова кликабельны (кроме служебных: артикли der/die/das/ein, союзы und/oder/aber и т.п.). Логика: сначала поиск в словаре пользователя (мгновенно) → если нет, запрос к Gemini (~1 сек с индикатором) → новое слово автоматически сохраняется в словарь со статусом «встреченное». Встреченные слова текущего урока кэшируются в L.encounteredCache для быстрого повторного поиска.

- Онбординг-подсказки (v41): система одноразовых оверлеев при первом входе на экран. Флаги хранятся в localStorage (ключи hint_shown_*). Закрываются кнопкой «Понятно ✓» или тапом по фону. Покрыто 6 экранов: новые слова урока, задания (drill), квест, суперквест, словарь, главный экран (прогресс-кольцо). Тексты подсказок в объекте HintTexts.

- Приоритет модели Gemini (v43): порядок перебора изменён — `gemini-2.0-flash` теперь первый (быстрее 2.5-flash в ~3 раза при сопоставимом качестве для A1). Порядок: `2.0-flash → 2.5-flash-lite → 2.5-flash → 2.0-flash-lite → 1.5-flash → 1.5-flash-8b`. Решение: скорость ответа важнее максимальной точности модели для контента уровня A1–B1.

- Параллельный запрос при старте суперквеста (v43): запрос `functional_new` к Gemini запускается через `Promise` сразу при входе в `startSuperLesson()`, не блокируя подготовку данных. Сокращает время до первого экрана суперквеста.

- Фикс попапа тап-слова — пустые поля (v43): `lookupAndShowWord` теперь проверяет полноту записи в словаре (`translit || translation`). Если запись есть, но поля пустые — идёт к Gemini, получает данные и дополняет существующую запись. Ранее попап показывал только слово + 🔊 для записей с пустыми полями.

- Дополнение словарных записей (v43): при тап-запросе к Gemini (`translate_word`) — если слово уже в `DB.vocab` с пустыми `translit`/`translation` — поля дополняются (не перезаписываются), `Storage.save()` вызывается только при фактическом изменении.

- Фикс обрыва сети в суперквесте (v43): при сбое соединения на сцене 2+ (не первой) вместо немедленного `finishSuper()` показывается экран «Связь прервалась» с кнопками «Повторить сцену» и «Завершить Суперквест». Пользователь сам решает продолжать или нет. `finishSuper()` при обрыве на сцене 1, акте 1 по-прежнему не вызывается — показывается экран «Нет связи» с повтором старта.


- **Чистка списка моделей Gemini — критический фикс (v44):** в коде были прописаны мёртвые модели, отключённые Google. `gemini-2.0-flash` и `gemini-2.0-flash-lite` отключены 01.06.2026; `gemini-1.5-flash` и `gemini-1.5-flash-8b` — архивированы/отключены (404). В v43 `gemini-2.0-flash` по ошибке стояла ПЕРВОЙ в очереди → приложение упиралось в мёртвый эндпоинт, перебирало дохлые модели с паузами и ретраями → отсюда загрузка по 10-15 секунд. Новый список — только живые модели, порядок по лимитам Free Tier: `['gemini-3.1-flash-lite', 'gemini-3.5-flash', 'gemini-2.5-flash']`.
  - `gemini-3.1-flash-lite` — первая: 30 RPM / 1 000 000 TPM / 1500 RPD. Самые щедрые лимиты Free Tier, в 429 упереться почти невозможно.
  - `gemini-3.5-flash` — фолбэк: 15 RPM / 250k TPM / 1500 RPD. Умнее, без объявленной даты отключения.
  - `gemini-2.5-flash` — последний фолбэк: 10 RPM / 250k TPM / 1500 RPD. EOL 16.10.2026 — потом удалить.
  - Рассмотренные и отклонённые альтернативы: алиас `gemini-flash-latest` (рекомендация шпаргалки — не устаревает, но непрозрачно какая модель отвечает, хуже для отладки методики); `gemini-2.5-flash-lite` (лимиты не указаны/ниже, чем у 3.1-flash-lite — смысла нет).

- **maxOutputTokens — решено НЕ ставить (v44):** проведён расчёт расхода токенов. Обычный урок A1 ≈ 11 500 токенов / ~19 запросов; суперквест (30 сцен) ≈ 56 000 токенов / ~61 запрос; средний день пользователя (2 урока + 1 суперквест) ≈ 80 000 токенов / ~99 запросов. Против лимитов 3.1-flash-lite: RPD 99/1500 = 6.6%, дневной расход токенов = 8% от ОДНОминутного лимита TPM. Запас огромный. Лимит maxOutputTokens создал бы только риск обрезанного JSON — оставлен без ограничения.

- **Анализ узкого места — RPM, не токены (v44):** реальное (и единственное) ограничение для AILEX — частота запросов в минуту (RPM), а не объём токенов и не дневной лимит. Суперквест делает ~61 запрос; при быстрой игре (сцена за 20-30 сек) можно превысить 30 RPM у 3.1-flash-lite или 10 RPM у 2.5-flash → 429. Решается выбором lite-модели с 30 RPM первой + существующей логикой пауз при 429 (ждём 2с→4с, повтор той же модели до 3 раз, иначе следующая модель). Логику ретраев менять не потребовалось.

- **Фикс UX упражнения «Слово в контексте» (v45):** по результатам тестирования выявлено три проблемы в `drillContext`:
  1. **Перевод предложения** — под предложением с пропуском теперь всегда показывается перевод на родной язык (`📝 ...`) из поля `item.prompt_native`. Ранее перевода не было, пользователь видел только слово-подсказку без контекста всей фразы.
  2. **Двухуровневая лампочка** — автоподсказка для «трудных» слов (`hardWord`) убрана полностью (показывалась сразу, не давая попробовать самому). Теперь: первый тап 💡 → скелет (первые 2 буквы) + транскрипция; второй тап → полное слово + транскрипция + перевод. Отклонено: одноуровневая подсказка (не даёт прогрессивного раскрытия).
  3. **TTS в «Фразах на каждый день»** — кнопки 🔊 не срабатывали из-за конфликта механизмов: `drillPhrases` использовал `data-t` + querySelectorAll('.tts'), тогда как везде в приложении используется `data-say` + `onclick="replaySay(this)"`. Исправлено унификацией на стандартный механизм.

- **Кэш незавершённого урока (v45):** добавлен объект `LessonCache` (save/load/clear). Весь объект `L` (слова, задания от Gemini, текущий шаг) сохраняется в localStorage при каждом переходе между шагами. При релоде/выходе из приложения урок восстанавливается мгновенно без обращения к Gemini. В главном меню появляется кнопка **▶️ Продолжить урок X.X.XX** (Задание N из M) если есть незавершённый урок не старше 24 часов. Кэш очищается при завершении урока (`finishLesson`) или при старте нового урока с другим кодом. Ключ кэша включает версию приложения (`AILEX_lesson_cache_v45`) — при обновлении приложения старый кэш автоматически игнорируется. Объём кэша: ~10-15 КБ на урок. Суперквест не кэшируется (сложная структура с промисами). Отклонено: восстановление суперквеста (слишком сложная сериализация state machine с параллельными запросами).

- **Каркас на вопросах-выборах + точность грамматики в фидбэке (v53, QUEUE-32):** на задании-выборе («ручной режим или автоматику?») scaffold выдавался как «Ich nutze den ___» — артикль den вынесен за прочерк, подходил только для der Handbetrieb; для die Automatik навязывал неверное «den Automatik» и выдавал род (открытый выбор превращался в ложный). Плюс check_answer в разборе приписал Automatik средний род (на деле женский). Правки: (A) в промптах quest_step и superquest — при выборе из слов разного рода/падежа артикль НЕ выносится за прочерк, входит в прочерк вместе с существительным («Ich nutze ___.»); видимая часть каркаса одинаково верна для любого варианта. (B) в check_answer квеста и суперквеста — explanation/rule_* обязаны быть фактически верны (род/падеж/спряжение), при задании-выборе любой вариант с грамматически верным артиклем = correct=true. Отклонено: чинить каркас на стороне кода (срезать определитель перед ___) — хрупко, род приложению неизвестен → слой промпт.

- **Слова задания с артиклем рода (v53, QUEUE-33):** в «📒 Слова задания» существительные показывались без артикля (учащийся не знает род → не образует падеж). Правка C: в answer_glossary и encountered_words промптов quest_step/superquest поле term для существительных целевого языка обязано включать артикль рода в словарной форме (die Automatik, der Handbetrieb); прилагательным/наречиям/глаголам — без артикля; в безартиклевых языках — без артикля. Компаньон в коде: совпадение в кэше тап-перевода (lookupAndShowWord) сделано артикль-терпимым (срезается ведущий der/die/das с обеих сторон) — тап по «Automatik» мгновенно находит «die Automatik» без досыла к Gemini.     - *Обычный квест* (шаг внутри урока): позиция сцены хранится в `L.qResume` (scene/summary/choice/errors/branchesUsed), сохраняется в снимок урока на каждом переходе, восстанавливается при входе в `runQuest`, очищается по завершении квеста. Возврат — через существующий пункт «Продолжить урок».
    - *Суперквест* (отдельный многоактный режим, раньше не кэшировался вообще): позиция (акт `L.actIndex`, сцена `L.sqScene`, ветки, новые слова, сквозная сводка `L.superStory.summary`, арка `L.superStory.arc`) сохраняется через `LessonCache` перед каждой сценой (`saveSuperPos`). Новый пункт меню «Продолжить Суперквест» (глава/сцена) и `resumeSuper()` восстанавливают `L` и перезаходят в сохранённый акт (`L._resumeScene`). Кэш чистится при старте нового Суперквеста и в `finishSuper`.
    - *Архитектурный компромисс (согласован):* сцены генерируются Gemini по одной из сводки/арки, поэтому при возобновлении текущая сцена перегенерируется — сюжетная непрерывность сохраняется, точная формулировка прерванной сцены может слегка отличаться.
    - *Ограничение:* единый слот кэша (`LESSON_CACHE_KEY`) — одновременно хранится один незавершённый эпизод; старт урока или Суперквеста сбрасывает прежний незавершённый. `LESSON_CACHE_KEY` не бампался: новые поля аддитивны и защищены проверками, кэш v61 совместим.

- **QUEUE-49 (v63): утечка кэша урока между профилями.** Снимок незавершённого урока лежал в едином слоте без привязки к профилю — переключение профиля могло подставить чужой урок. Фикс: в каждый снимок `LessonCache` пишется `_profileId: Storage.activeId()`; при чтении несовпадение профиля возвращает `null`, снимок при этом НЕ удаляется (чужой профиль свой прогресс не теряет). Миграция: снимки v62 без `_profileId` после обновления невидимы и истекают штатно в течение 24 часов. *(Запись восстановлена 17.07.2026 — сессия v63 не была задокументирована.)*

- **Интервальное повторение слов + сюжетность развилок (v65, QUEUE-55 + QUEUE-48, спека 4.27).** Поверх v64 (Ядро).
  - **QUEUE-55 — интервальное повторение 1/3/7/30 (спокойный слой).** Добавлен второй слой повторения поверх существующего срочного. Выученное слово получает ступень `srs_stage` (0–4) и дату показа `srs_due` (числовой номер дня — `today()` в формате ДД.ММ.ГГ хронологически не сравнить; `date_review` не переопределяется). Лестница: выучено → показ через 1 → 3 → 7 → 30 дней; верно → ступень выше, ошибка → сброс на день 1 + в `reviewQueue`; вершина (30 дней) → «освоено», снят с графика. Модуль `SRS` (`SRS_GAP=[1,3,7,30]`). Recap наполняется: срочный слой (трудные + импорт/интенсив) → подошедшие по дате (самые просроченные первыми) → срез 7; прежний добор «последние 10» упразднён. Суперквест тоже на графике: подошедшие по дате слова идут первыми, ступени двигаются по итогу (активно требуемые `must_use` вверх при 0 ошибок; ошибка на подошедшем слове — сброс). Gemini не задействован — чистая логика приложения, `generate_review_sentences` без изменений.
  - **«Освоено» — только вершина лестницы (методическая правка).** Старое авто-«освоено» (0 ошибок за один урок) убрано — это оперативная память, не долговременная. Выход из `reviewQueue` (QUEUE-54, два чистых урока) больше НЕ ставит «освоено», а возвращает слово на спокойную лестницу. Единственный вход в «освоено» — прохождение 30-дневной ступени. Затронуты `finishLesson` и `finishSuper` (оба места выхода из очереди).

Языковое задание (`expected_target`) отделено от выбора ветки. Промпт-правило (как QUEUE-34/36) — 100% гарантии нет, проверяем на тесте.
  - **Миграция:** старый словарь массово на лестницу не переносится — слова входят в спокойный слой по мере касания повторением; исторически трудные идут через срочный слой. Backfill — на будущее по итогам теста.

Тап-перевод распространён со сцен Квеста/Суперквеста на задания урока и разбор ошибки; на кнопки-варианты Recognition тап не вешается (конфликт с выбором ответа).

- **Сохранение незавершённого занятия — снимок без срока годности (v67, спека 4.29).** Пакет из пяти связанных фиксов; повод — репорт: 20 заданий из 21 сделано, прервался на квесте, вернулся — урок с нуля.

Слот раздельный по профиль+тип+код занятия (`lesson` / `a0` / `super`) — старт Суперквеста больше не стирает незавершённый урок. До 5 эпизодов на профиль, вытеснение самого старого по дате касания. `LessonCache` остался тонкой обёрткой над `Episodes` (save/hydrate/lesson/clearLesson/clearSuper), точки вызова в коде не переписывались.
  - **Инвалидация через 24 часа убрана полностью.** Снимок живёт до завершения занятия или до явного перезапуска этого же занятия. Обоснование в спеке: занятие раз в два-три дня — нормальный режим, любая граница по времени обнуляет ровно того, кто занимается реже.
  - **Уроки A0 получили сохранение (раньше не сохранялись вообще).** Позиция жила в локальной переменной `stepIdx` внутри замыкания `startA0Lesson`, `LessonCache` не трогался, а кнопка «Продолжить» в меню явно исключала A0 (`!isA0 && !isExam`) — выход на любом шаге, включая финальную мини-сцену, терял урок целиком. Прохождение вынесено в `runA0Lesson(S)`; в снимок пишется позиция **и содержимое урока** (буквы, 12 упражнений, мини-сцена) — урок A0 генерируется одним запросом, повторная генерация вернула бы другой урок. Новый пункт меню «Продолжить урок A0.1.NN · Шаг N из M».
  - **Ключ хранилища зафиксирован навсегда**, версия схемы (`EPISODES_SCHEMA`) перенесена внутрь снимка. Прежний ключ нёс номер билда (`AILEX_lesson_cache_v46`) — каждое обновление приложения убивало начатый урок у всех пользователей. Разовая миграция старых ключей в эпизоды при загрузке (`Episodes.migrate()` в `boot`).
  - **Обрыв связи в квесте больше не завершает урок.** Было: `toast('Связь прервалась — квест засчитан') → cfg.onDone() → finishLesson()` — снимок стирался, `nextStage` сдвигался, то есть возврат к занятию после сна телефона (сеть поднимается не сразу) стоил всего урока. Стало: экран «Связь прервалась» с кнопками «Повторить сцену» / «Выход в меню», позиция сохранена — по образцу Суперквеста (v43). Позиция квеста теперь пишется в `saveQuestPos()` при входе в каждую сцену (единая точка), а не только при переходе дальше — выход на первой сцене квеста больше не теряет её.
  - **Переполнение хранилища не молчит.** Раньше неудачная запись снимка проглатывалась `catch(e){}` — кнопка «Продолжить» просто не появлялась. Теперь при нехватке места старые эпизоды уступают текущему, а при повторной неудаче — предупреждение пользователю.
  - **Чистое время занятия:** в снимок пишется накопленное время (`_spent`), при возврате `t0` сдвигается — пауза между заходами не попадает в `duration_sec` аналитики.
  - **Экзамен сознательно НЕ сохраняется** (решение 23.07.2026): пауза посреди замера даёт время найти ответы, результат перестаёт отражать уровень. Прерванный экзамен начинается заново.
  - *Ограничение:* при возврате внутрь мини-сцены A0 или сцены квеста занятие открывается с начала этого шага (сцена квеста перегенерируется — компромисс v62 сохранён).

# 1.1. Спека 4.26 — Ядро языка (подписана 17.07.2026, кода ещё нет)

**Повод.** Тест A1.1.11 на v63: урок выдал 3 слова вместо 7, без единого глагола. Разбор кода: `generate_vocab` присылает 7, фильтр дубликатов режет до 3, добора нет, порог `words.length < 3` пропускает обеднённую порцию молча. Корень — в `known_vocab` уходят только последние 300 слов Словаря (`known.slice(-300)`), остального модель не видит и честно предлагает повторно. Чем больше словарь (у Constantin'а импорт ЧПУ + 11 уроков A1), тем чаще пустой ход.

**Решение (архитектурное).** Выбор слов переходит приложению. Gemini генерит Ядро языка один раз при создании профиля; порцию урока приложение собирает из Ядра арифметикой, без запроса. `generate_vocab` из штатного пути урока уходит в резерв.

**Принятые решения (все — 17.07.2026):**
- Ядро генерится на устройстве, в локальную базу. Файлов на GitHub нет (пользовательские данные + 72 языка не сопроводить).
- Закрытые классы (местоимения, предлоги, союзы, частицы, ~200 неправильных глаголов) — целиком при создании профиля. Открытые — по уровням: A1 (~600 слов) при создании, следующий уровень фоном.
- Два слоя: общий (от интересов не зависит) + тематический 100–200 слов (перегенерится при смене интересов).
- Порция: **5 новых + 2 неправильных глагола, гарантированно каждый урок** (норма спеки сохранена, но стала гарантией вместо «если уместны»).
- Смена интересов: невыданные слова старой темы → архив (из порции выбывают, по тапу в контексте работают); выученное остаётся в Словаре.
- Показ форм ≠ требование форм: формы неправильного глагола показываются справочно с первого дня; в заданиях требуется только пройденное (`known_rules`).
- Ядро человеку списком не показывается. **Таблица неправильных глаголов отвергнута** — формальная зубрёжка; вместо неё «голограмма» (слово в разных контекстах).
- Выход из очереди повторения: два урока подряд без ошибок → «освоено». До 4.26 очередь чистил только Суперквест и она росла монотонно.
- Миграция существующих профилей: Ядро догенерится разово при первом запуске v64, экран ожидания, накопленный Словарь уходит в `exclude`.
- Интенсив (импорт ЧПУ) Ядра не касается — работает как работал.
- Ядро исчерпано / сбой генерации → резерв `generate_vocab` + добор до 7 повторным запросом.

**Найденный попутно баг (в v64).** Экран новых слов проверяет поле `irregular_forms`, а контракт `generate_vocab` возвращает формы в поле `forms` — условие не выполняется никогда, **формы на экране новых слов не показывались ни разу**. В Словарь они попадают (`w.forms || w.irregular_forms` при сохранении). Правится одной строкой, от Ядра не зависит.

**Отклонено:** файл-словарь на GitHub; ссылка на словарь в запросе (модель в наших вызовах в интернет не ходит); загрузка полного словаря языка (300k+ слов, 99% не нужны на A1); готовые файлы ядра в репозитории.

**Допущение (записано честно):** частотность в Ядре приблизительна — Gemini не считает корпус, а воспроизводит то, что «помнит» о частотных списках. Верх ядра (закрытые классы, топ-200 глаголов) добротен; ранг в глубине — примерный. Для костяка достаточно.

# 1.2. Спека 4.28 — «Собери фразу» + тап-перевод во всех заданиях (реализована в v66)

**Повод.** Тест A1.1.11 на v65 (скриншоты 21.07.26) вскрыл в Construction (`drillConstruct`) три болячки: (1) смешанный язык в условии — русский текст с непереведёнными целевыми словами («Мне нужен новый рюкзак для Weltreise»), в пределе весь эталон целиком в поле условия («Die Kontrolle läuft bei der Produktion.» как «задание»); (2) готовый ответ выдан прямо в условии; (3) неизученные слова в задании (Werkstor, Weltreise). Корень (1)/(2) — условие рисуется из `prompt_native` (строка рендера `«${prompt_native}»`), а Gemini кладёт туда целевые слова/эталон вопреки промпт-правилу симметрии (природа QUEUE-09/34).

**Решения (все — 21.07.2026, подписаны):**
- Новый основной тип Construction на A1 — **«Собери фразу»** (word-bank): перемешанный набор всех слов эталона + перевод-условие на родной; учащийся **печатает сам** (не тап по плиткам — печать намеренна; плитки «учат медленно»). Свободный ввод остаётся дополнительным вариантом.
- **Набор формирует приложение** из `expected_target` (разбить + перемешать) — логика, не Gemini. Гарантии кодом: все слова в наборе (включая служебные), целевой фразы в условии нет вовсе.
- **Чистое условие + квота** (правки контракта `construct`): `prompt_native` — 100% родной без единого целевого слова; `expected_target` — из известных слов, **≤1 нового** на фразу; `glossary` — все слова эталона, включая служебные, с переводом и транскрипцией.
- **Тап-перевод во всех заданиях урока** + на экране разбора ошибки: жест распространяется со сцен Квеста/Суперквеста на «Слово в контексте», «Собери фразу», свободный ввод, повторение в контексте, «Фразы на каждый день», разбор ошибки. Переиспуются `tapAllWordsHTML`/`attachTapAll`/`lookupAndShowWord`. Тап ничего не выдаёт раньше времени.

**К проверке на тесте:** тап на вариантах Recognition конфликтует с их основным назначением (тап = выбор ответа) — в v66 на кнопки-варианты тап НЕ вешаем, отметить отдельно. Полная спека: docs/AILEX_4_28.md, разделы 5.2, «Структура 21 задания» (ред. 4.28).

**Отклонено:** тап по плиткам вместо печати; построчная код-сверка слов условия (хрупко) — заменено архитектурой «набор строит приложение».

# 8. Базовые принципы (рамка, неизменно)

- Логика → приложение, контент → Gemini, хранилище → локальная база на устройстве.
- Смена версии модели Gemini НЕ меняет методику обучения.
- Gemini — только генератор контента: слова, фразы, проверка, квесты — всегда строго в JSON по контракту. Методику не знает и не определяет.
- Принцип терминологии: методические термины в интерфейсе — на родном языке учащегося.
